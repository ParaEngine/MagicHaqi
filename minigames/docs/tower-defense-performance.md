# 细胞免疫塔防性能方案

## 目标

`minigames/haqi_pet_tower_defense.html` 以稳定 60 FPS 为目标。战斗逻辑覆盖高密度敌人、子弹、状态效果、护膜恢复、持续伤害、宠物动作和 GUI 反馈动画，性能方案按完整塔防规模设计。

## 帧预算

60 FPS 下单帧总预算约 16.7 ms。建议预算如下：

| 模块 | 预算 |
| --- | ---: |
| 逻辑更新 | 3 ms |
| 目标选择和命中判定 | 2 ms |
| 动态绘制 | 7 ms |
| HUD 和 DOM 写入 | 1 ms |
| 音频触发 | 0.5 ms |
| 余量 | 3.2 ms |

移动端低端机按 45 FPS 保护线处理：当连续多帧超过预算时，优先降低粒子、发光、阴影、浮字密度，不能降低输入响应和核心逻辑正确性。

## 架构

游戏采用 Canvas + SVG + DOM 的混合架构：

- Canvas 负责动态战场：防御塔、攻击范围、伤害倍率标签、路线徽标、病毒、子弹、宠物基地、生命条、护膜条、状态图标、状态环、爆炸、粒子、浮字、金币飞行、白细胞和变异场。
- SVG 负责静态场景和低频交互层：背景、血管路径、细胞器背景和建造点。
- DOM 负责 HUD、按钮、弹窗、难度选择、升级分支选择和进化选择。

`renderAll()` 的主路径是 `renderCanvasWorld()`。它每帧清空 Canvas 并批量绘制动态实体，不在 RAF 中创建敌人、子弹、塔、基地、状态环和特效 SVG 节点。

## 设计约束

动态实体不能在 RAF 中创建 DOM/SVG 节点。以下内容必须走 Canvas 绘制管线：

- 攻击特效、命中特效、受击动画、病毒变形、宠物动作和 GUI 战场反馈。
- 生命条、护膜条、状态图标、状态环、伤害倍率标签、路线徽标、选中范围和变异场覆盖效果。
- 粒子、浮字、金币飞行、白细胞冲刺等短生命周期对象。

SVG 只用于静态或低频交互对象。建造点可以保留为 SVG，因为它们数量固定，并且只在布局或建塔状态变化时重建。

## 逻辑更新约束

### 敌人防御

精英病毒维护 `hp/maxHp`、`defenseMode`、`shield/maxShield`、`baseArmor/armor`、`lastDamagedAt` 和 `effects`。`defenseMode` 控制同一只精英病毒只表现为护膜型或外壳型。护膜恢复在 `updateEnemies()` 内按 `dt` 增量计算，只更新数值，不创建额外对象。

护膜恢复规则：

- 受伤会刷新 `lastDamagedAt`。
- 未受伤超过恢复延迟后，护膜按最大值比例恢复。
- 恢复数值夹在 `0..maxShield` 内。

### 状态效果

持续伤害使用敌人实例上的 `effects` 字典。每个效果包含类型、来源、持续时间、tick 间隔、下次 tick 时间、每秒伤害、层数、软化外壳值和绕过护膜标记。

状态更新规则：

- `updateEnemyEffects()` 在敌人移动前处理过期与 tick。
- 同类状态刷新持续时间，层数存在上限。
- DOT tick 走统一 `damageEnemy()` 结算入口。
- 毒素标记通过 `enemyHasShieldBypass()` 影响该目标承受的所有伤害。
- 热灼反应通过 `effectiveEnemyArmor()` 降低该目标的有效外壳值。

状态效果不创建 DOM/SVG 节点。热灼反应和毒素标记用血条上方的小型 Canvas 图标表达，持续伤害 tick 不生成状态文字浮字。

### 炮塔分支

2 级、3 级和 4 级分支通过塔实例字段 `tier2Branch`、`tier3Branch`、`tier4Branch` 驱动：

- 远端感知和广域感知只影响 `towerRange()` 的射程计算。
- 分裂弹幕在射击时概率追加第二发子弹。
- 代谢加速只影响 `towerFireInterval()` 的冷却计算。
- 热灼反应和毒素标记在 `hitEnemy()` 命中时写入目标 `effects`。
- 溅射子弹继承状态类型，并按较低系数写入目标状态。
- 路线徽标通过 Canvas 文本和圆角底片绘制在防御单元下方，不创建 DOM 节点。
- 自身伤害倍率标签通过 Canvas 文本和圆角底片绘制在防御单元下方，不创建 DOM 节点。

## 性能地基

1. 建造点使用 dirty 渲染，仅在布局和建塔状态变化时重建。
2. HUD 使用缓存，只在显示值变化时写 DOM。
3. 塔按钮、升级按钮和分支按钮只在 HUD 更新路径写入状态。
4. 敌人维护 `x/y` 坐标缓存和 `enemyById` 索引，减少 `pointAt()` 和数组查找。
5. 子弹命中使用 `enemyById.get()`，范围伤害使用敌人坐标缓存。
6. 护膜、外壳、状态效果和持续伤害都挂在敌人对象上，避免额外全局数组扫描。
7. 防御单元路线挂在塔对象上，路线徽标和伤害倍率标签随 `drawTowersCanvas()` 一起绘制。
8. FX 更新使用原地更新和过滤，减少临时对象分配。
9. 粒子、爆炸、浮字、金币飞行设置上限，避免特效瞬间堆积导致卡顿。
10. 音效触发节流，避免密集射击和命中时创建过多 WebAudio 节点。
11. Canvas 使用 DPR 上限和 `setTransform()` 保持逻辑坐标稳定。
12. 塔点击使用坐标拾取，不依赖塔的 SVG 节点。

## 特效预算

| 类型 | 常规上限 | 降级上限 |
| --- | ---: | ---: |
| 敌人 | 80 | 120 |
| 子弹 | 120 | 160 |
| 粒子 | 260 | 120 |
| 爆炸圈 | 90 | 40 |
| 浮字 | 80 | 35 |
| 金币飞行 | 60 | 24 |
| 白细胞 | 20 | 12 |

降级策略按顺序执行：减少粒子数量、缩短粒子寿命、关闭发光和阴影、减少浮字、降低非关键 GUI 动画频率。状态环、护膜条和生命条属于核心战斗反馈，在常规降级中保留。

## 可复核检查

1. 打开 `minigames/haqi_pet_tower_defense.html`，确认 `renderAll()` 的动态路径调用 `renderCanvasWorld()`。
2. 检查 `updateHud()`，DOM 文本、按钮隐藏状态和 class 只在缓存值变化时写入。
3. 检查 `updateShots()`，子弹目标查找使用 `enemyById`。
4. 检查 `updateEnemies()`，敌人移动、状态 tick 和护膜恢复在同一敌人循环中完成。
5. 检查 `damageEnemy()`，直接命中、持续伤害、白细胞和溅射都走统一结算入口。
6. 检查 `updateFx()`，爆炸、浮字、金币飞行和粒子为原地更新。
7. 检查动态战场绘制由 `renderCanvasWorld()` 内部的 `drawTowersCanvas()`、`drawEnemiesCanvas()`、`drawShotsCanvas()`、`drawHeroCanvas()` 和 `drawFxCanvas()` 完成。
8. 检查塔选择通过 `towerAtPoint()` 坐标拾取完成。
9. 在挑战难度跑到后期波次，观察射程路线、分裂弹幕、持续伤害、护膜恢复和白细胞并发时的输入响应。
10. 用开发压力场景模拟 80 个敌人、120 个子弹、260 个粒子、80 个浮字，确认逻辑和输入响应稳定。

## 完成标准

- 普通移动设备在常规波次保持 58-60 FPS。
- 高压波次短时掉帧后能快速恢复，不出现 GC 周期性大卡顿。
- 点击建造、升级、选择分支、释放白细胞、选择进化无明显输入延迟。
- 攻击特效、受击动画、病毒变形、状态环和护膜反馈接入统一 Canvas 与 FX 预算。
- RAF 中不高频创建动态 DOM/SVG 节点。
