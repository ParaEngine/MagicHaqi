# 哈奇矩阵破解玩法设计方案

## 1. 游戏定位

《哈奇矩阵破解》是一款矩阵连线路径解谜小游戏。玩家在一个方形破解面板上，从左上角起点连接到生成路线末端的终点，途中需要经过全部破解点，并利用道具节点处理障碍。游戏体验重点是基于当前矩阵状态判断哪些道具值得获取、何时使用、如何绕行或突破。

## 2. 核心目标

玩家需要完成以下目标：

1. 从起点开始连接路径。
2. 只能向上下左右相邻节点移动。
3. 路径不能交叉回绕。
4. 需要经过全部破解点，经过顺序不限。
5. 到达终点时，若全部破解点已接入，则通关。
6. 障碍点默认不可通过，需要绕行或借助道具处理。

## 3. 节点分类

### 3.1 基础节点

- 起点：固定在矩阵左上角，路径必须从这里开始。
- 终点：位于生成路线末端，完成全部破解点后到达这里即可通关。
- 普通节点：可直接通过，用于构成路径。
- 破解点：必须全部经过，顺序不限。
- 起点、终点、破解点使用不填充整格背景且不添加专属边框的样式，以带颜色的中心 SVG 图标作为主要识别信息。

### 3.2 障碍节点

- 障碍点：默认不可通过，可通过清除型道具移除，也可通过净化状态临时突破。
- 不可净化障碍：只在高级难度下出现，不能被净化效果突破，也不能被清除型道具移除，玩家只能绕行或借助关卡结构另寻路径。

### 3.3 道具节点

道具节点会放在可走路径上。玩家经过道具节点后会先获得对应效果。关卡不要求所有道具都必须使用，但道具配置应与当前障碍布局相关，具有实际价值。

道具节点分为两类：

- 主动触发型：经过后先存入当前持有栏，由玩家选择合适时机主动释放。适用于拥有非单格清扫能力的道具。
- 临时状态型：经过后立即进入短时生效状态，用于影响玩家下一次经过障碍时的结果。

当前道具类型：

1. 主动触发
- 脉冲：清除周围 8 格全部可清除障碍。
- 十字：净化当前节点上下左右各两格内的可清除障碍。
- 横切：打通当前整行上的可清除障碍。
- 纵切：打通当前整列上的可清除障碍。
- 顺旋：将周围8格的所有节点顺时针旋转。
- 逆旋：将周围8格的所有节点逆时针旋转。
- **主动触发道具可同时存在多个，随时使用，但只能使用一次**

2. 临时状态型
- 净化：获得一次突破能力，可直接穿过下一个普通障碍，但对不可净化障碍无效。
- **临时道具在普通拖拽过程中一旦触发即消耗；倒退到时间点时会恢复该时间点记录的矩阵与状态**
- **临时状态型道具会覆盖**

## 4. 关卡难度

难度由矩阵尺寸、基础路径长度、破解点数量、障碍密度、道具数量和道具池共同决定。所有难度都使用同一套路线形态约束和二维矩阵状态验算生成关卡。

- 入门：5x5，2 个破解点，1 个道具，主要出现脉冲和净化。
- 普通：6x6，3 个破解点，3 个道具，引入横竖线性清障与净化。
- 困难：8x8，4 个破解点，5 个道具，引入顺旋、逆旋与更复杂的主动道具组合，解法会实际使用旋转。
- 专家：9x9，5 个破解点，7 个道具，障碍更密集，并出现不可净化障碍，解法会实际使用旋转。
- 噩梦：11x11，7 个破解点，9 个道具，不可净化障碍占比更高，道具选择空间更复杂，解法会实际使用旋转。

## 5. 道具说明系统

当某个节点类型首次出现在当前玩家记录中时，游戏会弹出独立的新节点说明小窗；如果本局同时出现多个新节点，玩家可连续查看下一个。全部查看后，底部“节点”按钮会短暂闪烁，提示玩家可通过该入口重复查看节点说明。

底部“节点”按钮打开的是常规节点说明界面，用于回看已知基础节点和已解锁道具节点，不与首次出现的新节点说明共用弹窗流程。

说明界面包含：

- 起点、终点、破解点、障碍点。
- 已经见过的所有道具节点。
- 点击某一节点名称后显示该节点图案和详细作用。

## 6. 操作规则

- 鼠标或触摸拖拽连接节点。
- 只能连接上下左右相邻节点。
- 已经划过的路径不能通过拖回上一个节点来回退，路径不能交叉回绕。
- 破解路线显示在所有节点格子背景之上，节点中心图标显示在线路之上，避免线路被格子背景遮挡。
- 经过主动触发型道具后，界面需要显示当前持有道具，并允许玩家在合适时机主动释放。
- 当前持有栏按道具类型聚合同款道具，使用“×n”显示数量；持有栏设置最大高度，数量过多时在栏内滚动。
- 经过净化后，界面需要明确提示“下一次障碍突破有效”，直到该效果被消耗。
- 点击“倒退”会打开时间点列表，游戏按网格尺寸间隔记录全矩阵状态。每个状态点包含已绘制线路、所有节点、持有道具、净化状态、破解点状态和 canvas 缩略图。列表最多显示最近 10 个状态点，点击某个状态点后恢复该局到该状态，并清除该状态之后的记录。
- 点击“重开”重新生成当前难度关卡。
- 点击“下一局”生成下一关。
- 当页面 URL 带有 `vscode-livepreview=true` 时，底部操作区显示“自动破解”按钮；同一条件下可从浏览器控制台调用自动破解与批量测试接口。

## 7. 计分与通关

通关条件：

- 到达终点。
- 已经过全部破解点。

计分参考因素：

- 难度基础分。
- 破解点数量奖励。
- 时间消耗惩罚。
- 提示次数惩罚。

## 8. 代码设计

### 1. 图标

请给所有的节点设计一个独特的图标，确保玩家能够一眼识别不同类型的节点。图标设计应简洁明了，符合游戏整体风格。包括起点、终点、破解点、障碍点以及各种道具节点的图标。图标使用 svg 格式，方便在不同分辨率下保持清晰。不可直接用字母、数字当图标。起点、终点、破解点以中心图标为主，格子本身保持普通矩阵底色，不使用专属填充或专属边框。

### 2. 道具动画

请给所有的道具触发都设计一个动画效果，增加视觉反馈和游戏体验。
比如“脉冲”道具可以设计成一个扩散的圆形动画，清除周围障碍时伴随闪光效果；“十字”道具可以设计成向上下左右短距离发射的十字光束动画，被击中的障碍伴随爆炸效果；顺旋和逆旋道具需要让周围节点按对应方向移动并旋转；“净化”道具可以设计成一个短暂的光环动画，提示玩家下一次障碍突破有效。

### 3. 关卡生成

关卡生成需要按以下规则设计生成：
1. 确保起点和终点之间存在至少一条可行路径。
2. 随机分布破解点，确保它们在路径上。
3. 生成大量障碍点，允许封锁可行路径。
4. 生成道具节点，道具节点是游戏的核心。道具不一定能解决每个障碍，但每个障碍至少有一种道具能解决。
- 在障碍点封锁可行路径的前提下，必须实现一个 可行路径上有道具 - 道具能解决可行路径上的障碍 - 清除障碍后能够获得其他道具 - 其他道具能解决可行路径上的障碍 - 最终到达终点 的循环
- 但玩家有可能因为道具获取顺序的路径规划错误而无法通关，而不是只要照着道具一路获取就能通关。

算法实现关键点：
- 先生成一条从左上角出发的自避基础路径，路径长度按难度区间取值；路径会同时约束边界占比、早期贴边比例、转向比例与覆盖区域，避免生成外圈蛇形或单侧堆积的路径。
- 在基础路径上按顺序切分多个道具链段，每个链段由“可先到达的道具节点”和“后续阻断路径的障碍点”组成，保证玩家必须先获得或保留合适道具，才能继续接入后续路径。
- 困难及以上难度会围绕关键阻断点生成侧向封锁，保留道具节点侧和后续路线侧的连接，让阻断点更像一道需要处理的门，而不是可以从旁边绕过的单格障碍。
- 主动道具可以进入持有栏并同时存在多个。获得主动道具后，道具节点立即变成普通节点；释放后该道具从持有栏移除，不能再次使用。
- 倒退按记录的时间点恢复完整状态，不提供逐格拖回；高难度关卡因此可以更大胆地使用长路径、旋转阻断和多段道具链，因为玩家只能回到有限的记录点重新规划。
- 脉冲、横切、纵切按作用范围清除普通障碍；十字只处理当前节点上下左右各两格，不等同于横切加纵切；顺旋和逆旋不清除障碍，而是将当前节点周围 8 格的节点内容按对应方向旋转，用位置变化解除或制造阻断。
- 额外障碍生成时按分区和距离评分挑选候选格，优先放在远离已有障碍且分区较少的位置；困难及以上难度允许额外普通障碍作为地形压力分散出现，不局限在少数道具作用范围内，避免障碍集中堆积在同一块区域；不可净化障碍只在高级难度生成，用于压缩绕行空间，但不作为必须被道具清除的阻断点。
- 每局从当前难度道具池中抽取本局道具子池，再从子池中生成道具节点；同款道具可以生成多个。
- 困难、专家、噩梦的道具子池包含旋转类道具，且关卡验收要求实际解法会使用旋转。
- 每次生成后使用真实二维矩阵状态模拟整条参考解法，逐步验证节点类型、持有道具、净化状态、旋转后的节点分布、破解点集合与终点到达条件；不满足当前难度数量、形态和解法约束的关卡直接重生成。

### 4. 自动破解与测试

- 自动破解只在 `vscode-livepreview=true` 条件下开放，运行期间锁定手动交互。
- 自动破解沿参考解法推进，并对当前矩阵状态做实时试算：主动道具释放、净化消耗、顺旋/逆旋后的节点变化都按真实矩阵结果继续决策。
- 浏览器控制台可调用单局自动破解、单难度批量测试和全难度批量测试接口，用于验证当前实现与生成规则。
