联动一句话

一个班级配置一个由学校或教师拥有的 KeepWork 共享站点；每位学生在站点内拥有独立目录。教师发入口，学生完成行动，家长确认奖励，不再要求每个家庭另建站点。

三端职责

联动的重点不是让三方都进入同一个管理后台，而是让每个人只面对最少、最符合自身角色的操作。

🧑‍🏫

教师端：发布课堂认可

  • 在班级设置中配置一次学校共享 KeepWork 站点。
  • 单个添加或从 Excel 批量导入学生、学生账号和家长账号。
  • 按课堂表现加分，生成家庭成长奖励。
  • 通过“家庭入口卡”分别发送学生与家长链接。
  • 可查看家长处理结果；家长处理后不允许撤销奖励。
🧒

学生端：把进步带回家

  • 打开教师发送的专属链接。
  • 首次输入 KeepWork 账号和密码；密码不保存在页面中。
  • 后续会尝试恢复已登录会话，无需重复输入密码。
  • 只查看自己的宠物、课堂积分和待完成成长任务。
  • 完成小行动后，提交给家长确认。
🧑‍🧑‍🧒

家长端：决定家庭认可

  • 打开教师发送的家长链接，孩子会自动绑定。
  • 查看课堂奖励和孩子提交的家庭小行动。
  • 选择“计入成长”或“暂不计入”。
  • 家庭账本保留在家长侧，不会被课堂档案覆盖。
  • 仅当链接丢失时，才需要手动填写站点、班级和学生编号恢复绑定。

完整联动流程

以下流程描述的是一条课堂奖励从产生到家庭确认的实际生命周期。

0. 班级准备
教师在“班级设置”中填写年级、班级和学校共享站点路径，例如 teacher_zhang/class-growth。随后导入学生资料，并为每位学生配置其 KeepWork 用户名与家长账号白名单。
教师配置
1. 发布入口
教师在“家庭入口卡”中发布学生的授权与课堂档案，并分别发送学生入口和家长入口。链接已带上共享站点、班级和学生标识。
一键发送
2. 课堂加分
教师对某位学生加分。系统保留课堂积分流水，同时创建一条有效期默认 72 小时的家庭成长奖励。
奖励待办
3. 学生行动
学生在个人页面看到待办，例如完成学习小任务或分享当天收获；点击完成后，系统写入该奖励对应的学生行动记录。
孩子已完成
4. 家长确认
家长在“学校成长收件箱”看到孩子已完成的行动，选择是否计入家庭成长。该决定会生成回执，教师端可刷新查看状态。
家庭确认
5. 保留边界
课堂积分仍由教师端维护；家庭成长账本仍由家长端维护。教师的课堂档案只用于展示，不会覆盖家长自己的家庭记录。
各自留存
场景示例

“作业认真 +5”，不止停在课堂

老师为小雨加分后，宠物收到课堂认可；小雨回家完成“分享今天学到的一件事”；家长在收件箱确认后，这份努力才真正成为家庭成长记录。

共享站点、数据隔离与权限

所有联动文件都在一个学校共享站点内，但按学生编号分目录保存。这样减少建站成本，同时使每条奖励、回执和通知都能定位到对应孩子。

minigames/haqi-growth-link/v3/students/<studentId>/
  access-policy.json              # 学生与家长白名单、授权策略
  student-profile.json            # 当前学生的课堂档案
  events/index.json               # 该学生的课堂奖励索引
  events/<eventId>.json           # 单条课堂奖励
  actions/<eventId>.json          # 学生提交的家庭小行动
  receipts/<eventId>.json         # 家长“计入/暂不计入”的回执
  notifications/index.json        # 该学生的通知索引
  notifications/<notificationId>.json
应用层隔离所有 V3 文件带有 privacyScope: "student"，读取时会校验班级、学生编号、学生账号或家长白名单。
KeepWork 站点权限站点是否可读写仍由 KeepWork 的真实访问权限决定。教师需拥有编辑权限；学生和家长账号需要具备完成各自读取或回执写入所需的站点访问权限。

重要：白名单不等于 KeepWork ACL

页面中的家长白名单用于应用逻辑校验，不能替代 KeepWork 平台的真实成员权限。静态 HTML 无法自动创建站点、加入成员或授予编辑权限，这些操作必须先在 KeepWork 平台完成。

教师上线检查清单

  1. 已在 KeepWork 创建真实可访问的学校共享站点，格式为 用户名/站点名，不是任意多级路径。
  2. 当前教师账号拥有该站点编辑权限；能正常读取和写入页面文件。
  3. 班级设置已保存正确的学校共享站点路径。
  4. 每位需要参与联动的学生都已填写学生 KeepWork 用户名，且家长白名单至少有一个账号。
  5. 已发布至少一位学生的云端授权与课堂档案，并通过入口卡发送学生、家长链接。
  6. 已确认学生和家长账号具备 KeepWork 站点所需的访问权限。
  7. 用一个测试学生完成“教师加分 → 学生行动 → 家长确认 → 教师刷新回执”的闭环验证。

常见情况

为什么发布时出现 HTTP 404 或“请求不合法”？

通常表示填写的站点不存在、站点路径不符合 用户名/站点名 格式，或当前登录教师没有该站点的编辑权限。先在 KeepWork 中确认站点和权限，再重新发布。

学生每次登录都需要输入学生编号吗？

不需要。普通使用时学生只打开教师链接，再输入账号和密码；链接携带学生标识。学生编号仅在没有链接时用于手动恢复家长绑定。

学生或家长能看到其他孩子的数据吗？

应用会始终使用当前学生的独立目录，并校验策略、学生编号与账号信息。但对真实敏感数据，仍必须正确配置 KeepWork 站点的访问权限，不能只依赖页面白名单。

为什么家长暂时不能“计入成长”？

当前奖励可能尚未被学生提交家庭小行动、已过 72 小时有效期、已被撤销，或家长账号没有站点写入权限。家长端会显示相应状态或同步失败信息。

日常使用建议

教师把入口卡发一次即可；学生首次登录后由浏览器恢复会话，家长链接也会自动绑定孩子。日常只需在课堂加分、学生完成行动、家长确认三处完成最小闭环。
