# 蛋蛋星球 MagicHaqi —— 产品设计文档

## 1. 概览

- **应用名**: 蛋蛋星球 (MagicHaqi)
- **类型**: H5 虚拟宠物养成游戏 (类 Tamagotchi + Toca Life World)
- **目标平台**: 移动端优先 (竖屏 9:16)，兼容桌面端
- **目标用户**: 6–14 岁儿童及其家长（KeepWork 平台用户）
- **核心定位**: 用 AI 生成「会长大、会聊天、有记忆」的专属虚拟宠物

## 2. 核心玩法

### 2.1 宠物 DNA 系统（核心差异化 #1）
- 每只宠物拥有 12 位字符的 DNA 编码 (例: `A3B7-K2M9-X4Q1`)。
- DNA 决定宠物外观特征：物种、毛色、体型、配件、稀有度等。
- **繁殖**: 两只成年宠物可"结婚"，子代 DNA 由父母 DNA 通过分段交叉 + 低概率突变生成（类似遗传算法 crossover + mutation）。
- 首次养成可选「初始 DNA」(随机生成或输入种子) 并通过 AI 文生图生成专属外观立绘。
- DNA → Prompt 转换：DNA 字段映射到一组中文形容词，拼接成 `genImage` prompt。

### 2.2 养成循环（Tamagotchi 内核）
状态属性 (0–100): `体力 / 心情 / 清洁 / 亲密度`。
- 离线时间衰减 (基于 `lastSavedAt` 时间差)。
- 互动操作: 喂食、洗澡、玩耍、睡觉；AI 互动内容并入玩耍入口。
- 阶段成长: 蛋 → 幼年 → 青年 → 成年 → 长老（每阶段触发一次外观重生成）。

### 2.3 房间装饰 + 多场景（差异化 #2，类 Toca Life）
- 多个房间场景：卧室、厨房、浴室、客厅、花园（解锁制）。
- 每个房间网格化布局 (例如 8×6 格)，可放置家具/玩具/食物。
- 拖拽放置、移动、删除家具。
- 商店购买家具，使用游戏内金币（通过日常互动获得）。
- 宠物可在房间内自由"行走"（简单寻路 / 随机漫步动画）。

### 2.3.5 星球经营层（新增 Level 0 玩法）
- 宇宙层不只是外观预览，已承担星球经营入口：天气、星际拜访、UFO 访客、星象校准、里程碑。
- 星球等级由宠物数量、宠物阶段、状态、背包、摆放物、星球天数、游玩时长和金币综合计算，最高 Lv.10。
- 可建造 / 升级 4 类星球设施，均最高 Lv.3：天气塔、航天站、UFO 停机坪、观星台。
- 设施建造消耗金币 / 生物燃料；升级后增强对应功能，例如天气持续更久、冷却更短，星际拜访燃料更省，UFO 礼物更多，星象倍率更高。
- 天气包含雨云、晴光、季风。天气状态会显示在宇宙层和星球地表层，雨云还会在陆地生态中催生植物。
- 星际拜访消耗生物燃料，奖励金币并提高宠物心情 / 亲密度；生物燃料由星球地表层收集宠物便便获得。
- UFO 访客每日一次，奖励食物或家具到背包；停机坪等级决定奖励数量。
- 星象校准每日一次，启用当天 buff，并影响宠物衰减或属性：贪食之月、花园星座、明晰星轨、潮汐双月。
- 哈奇岛入口由星球等级控制：至少孵化 1 只宠物且星球 Lv.3 后开启说明页。说明页分页展示已告别宠物，成年宠物可通过成人礼飞回哈奇岛并写入哈奇手帐；《哈奇小镇》入口继续打开客户端下载弹窗。

### 2.4 AI 数字人对话（差异化 #3，付费功能）
- 普通用户：文字聊天（基于 `sdk.aiChat`），消耗免费额度。
- 付费用户：解锁语音对话（`DigitalHuman` 数字人接口），头像即宠物立绘。
- **每只宠物拥有独立记忆**: `personalPageStore` 的 `MagicHaqi` workspace 下 `pets/<petId>.memory.md`。
  - 记录: 宠物性格、生活事件、与主人的对话摘要、喜好。
  - 系统 prompt 注入 memory.md 内容，给 AI 提供持续记忆。
- 对话后自动追加要点到 memory.md（增量摘要）。

## 3. UI / 屏幕

| 屏幕 | 路径 | 说明 |
|------|------|------|
| 启动/登录 | `view_login.js` | Logo + 登录按钮（未登录态显示） |
| 宠物列表 | `view_petList.js` | 我的宠物卡片网格 + 「孵化新蛋」 + 「繁殖」入口 |
| 孵化页 | `view_hatch.js` | 输入 DNA 种子 / 随机 / 父母选择 → 调用 AI 生成立绘 |
| 主家/房间 | `view_home.js` | 主舞台：宠物展示、状态条、互动按钮、房间切换 Tab、装饰模式 |
| 宇宙/星球经营 | `level_planet.js` | 星球总览、设施建造升级、天气、星际拜访、UFO、星象、里程碑、哈奇岛入口 |
| 星球地表 | `level_field.js` | 陆 / 水 / 空生态，户外摆放、收集便便为生物燃料、承接天气视觉效果 |
| 商店 | `view_shop.js` | 家具 / 食物 / 装饰品购买 |
| 背包 | `view_inventory.js` | 已购物品，拖到房间使用 |
| 聊天 | `view_chat.js` | 文字聊天 + 付费语音 (DigitalHuman) |
| 宠物档案 | `view_profile.js` | DNA 详情、成长曲线、记忆查看 |
| 设置 | `view_settings.js` | 语言（默认中文）、音效、清除数据 |

## 4. 状态机
```
未登录 → 登录 → 宠物列表
                 ├─ 无宠物 → 孵化 → 主家
                 └─ 已选宠物 → 主家
主家 ↔ 商店 / 背包 / 聊天 / 档案 / 设置
主家 (装饰模式) ↔ 主家 (普通模式)
```

## 5. 数据模型 (PersonalPageStore workspace = `MagicHaqi`)

| Key / 文件 | 说明 |
|------|------|
| `user/profile.json` | 金币、生物燃料、解锁场景、设置、宠物 ID 顺序、当前选中宠物、星球天气 / buff / 事件 / 设施状态 |
| `user/inventory.json` | 用户共享背包 |
| `user/layouts.json` | 用户共享家具布局 JSON |
| `pets/<petId>.json` | 单只宠物完整数据 (状态、DNA、外观图URL、创建时间、家长DNA、当前房间) |
| 文件 `pets/<petId>.memory.md` | 宠物长期记忆（AI 摘要） |
| 文件 `pets/<petId>.chat.log` | 聊天历史（最近 N 条） |

## 6. 外部依赖

- KeepworkSDK 核心: 登录、`personalPageStore`、`aiChat`、`aiGenerators.genImage`、`speech`、`remoteLog`。
- `DigitalHuman` (付费功能，按需 `new DigitalHuman({ sdk, container })`).
- Tailwind CDN (`cdn.keepwork.com/keepwork/cdn/tailwindcss@3.4.16.js`).

## 7. 资源清单

- `assets/icons/`: 状态图标 (饥饿/心情/清洁等 SVG)。
- `assets/images/`: 房间背景 (5 张 PNG/WebP, 9:16)、商店家具图标。
- `assets/audio/`: BGM (轻松循环)、互动音效 (点击/喂食/笑声)。
- `assets/data/`: 默认家具表 `furniture.json`、DNA 字段映射表 `dna_traits.json`、商店物品 `shop_items.json`。
- AI 生成的宠物立绘走 `genImage` 远端 URL，不本地化。

## 8. i18n
- 仅中文（zh-CN）。`translations` 对象保留扩展点，不接入语言切换 UI（默认隐藏）。

## 9. 验收标准

- [ ] 未登录 → 点击登录 → 登录窗口正常弹出；登录后展示宠物列表。
- [ ] 孵化新蛋：输入或随机生成 DNA → 调用 `genImage` 成功 → 宠物保存到 PersonalPageStore 并出现在列表。
- [ ] 繁殖：选两只成年宠物 → 子代 DNA 含父母片段 + 突变 → 生成新立绘。
- [ ] 主家页面：5 项以上互动按钮可用，状态条随时间衰减；切换房间正常。
- [ ] 宇宙层：可建造天气塔 / 航天站 / UFO 停机坪 / 观星台，并能升级到 Lv.3。
- [ ] 星球天气：召雨 / 晴光 / 季风有解锁、冷却、持续时间和跨层视觉反馈；雨云会在陆地生态生成植物。
- [ ] 星际拜访 / UFO / 星象：资源消耗、每日限制、属性奖励、背包奖励和事件记录正确。
- [ ] 装饰模式：拖拽家具放置/移动/删除，布局持久化。
- [ ] 聊天：文字聊天可流式回复；聊天结束后 memory.md 被追加。
- [ ] 付费判定（mock 标记 `user.profile.isPaid`）下，聊天页显示「语音对话」按钮，点击实例化 DigitalHuman。
- [ ] 数据 reload 后保持。
- [ ] 移动端竖屏 (375×667 / 414×896) 无滚动条溢出。
- [ ] 全部文案中文。
