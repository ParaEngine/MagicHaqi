# MagicHaqi 故事编辑器使用指南

本文档面向《蛋蛋星球 / MagicHaqi》的故事创作者、老师、运营同学和开发维护者，说明如何从一个故事想法开始，选择宠物演员、编写提示词、生成互动故事、制作场景、预览试玩、保存分享，以及当现有资源不够时如何参考 [docs/dev_guide.md](dev_guide.md) 使用内容生产工具补充自己的场景和角色。

故事编辑器不是单纯写文本的工具。它生成的是可播放、可互动、可分享的移动端故事 JSON：每一幕可以有背景、粒子、音乐、旁白、角色对白和互动任务；玩家在播放时会看到宠物站在场景里说话，并通过轻拍、喂养、洗澡或小游戏推动剧情继续。

## 一、故事编辑器能做什么

故事编辑器适合制作：

- 宠物日记：记录玩家和宠物的一次日常陪伴。
- 小冒险：例如寻找水晶、修好飞船、帮助迷路伙伴回家。
- 活动故事：节日任务、星球活动、课程活动、品牌主题故事。
- 教育故事：责任、合作、分享、清洁、照顾情绪等儿童主题。
- 宠物邀请故事：用官方宠物、稀有宠物或玩家自己的宠物做主角。

一个完整故事通常包含：

- `actors`：故事演员，通常来自玩家宠物，也可以是官方配置宠物。
- `scenes`：多个场景幕，每幕有背景、标签、粒子、音乐和时间轴。
- `timeline`：每幕按顺序播放的旁白、对白和互动任务。
- `ending`：故事结尾文字。
- 保存路径：玩家故事保存在 KeepWork workspace 的 `stories/<storyId>.json`，列表索引保存在 `stories/index.json`。

## 二、进入故事创作

### 1. 从游戏内进入

登录游戏后，在主菜单或故事入口进入“故事创作”。进入后会看到故事列表：

- 已保存的故事会以卡片显示。
- 点击“新建故事”进入编辑器。
- 点击已有故事卡片可以继续编辑或播放。
- 分享按钮会生成可分享链接。
- 删除按钮会移除对应故事记录。

### 2. 本地开发访问

本项目是纯前端静态游戏。可以直接打开 [MagicHaqi.html](../MagicHaqi.html)，也可以用 Vite/Live Server 访问，推荐本地服务方式，能更稳定加载 SDK、图片和 JSON。

常见入口：

```text
MagicHaqi.html
MagicHaqi.html?story=pet-story/story_of_dragons.json
MagicHaqi.html?story=stories/my_story.json
MagicHaqi.html?storyFrom=xiaoming&story=my_story.json
```

其中 `story` 参数用于直接进入指定故事播放器。项目内示例故事在 [pet-story/story_of_dragons.json](../pet-story/story_of_dragons.json)。当前登录用户自己的草稿故事可以用 `?story=stories/my_story.json` 从自己的 workspace 读取；如果要打开别人分享的玩家故事，链接必须带 `storyFrom=<username>`，否则 `stories/my_story.json` 只会被当成当前登录用户自己的相对路径。故事列表里的分享按钮会自动生成 `storyFrom`。

## 三、编辑器界面说明

故事编辑器顶部有四个主要入口：

| 区域 | 用途 |
| --- | --- |
| 创作 | 选择演员、输入故事提示词、选择短故事/标准/长故事、调用 AI 生成。 |
| 预览 | 像绘本一样检查封面、每一幕、对白、互动和节奏。 |
| 试玩 | 直接进入真实故事播放器，测试玩家体验。 |
| 高级 | 用可视化时间轴或 JSON 文本精修故事。 |

顶部“保存”会把当前故事写入 workspace，并更新故事索引。AI 生成成功后也会尝试自动保存。

### 创作区

创作区包含三件事：

1. 选择演员。
2. 编写故事提示词。
3. 选择故事长度并生成。

演员来自当前账号的宠物列表。可以添加多只宠物，也可以给演员改显示名。第一只演员默认作为主角；点击演员卡可以调整主角。主角在故事 JSON 中会标记 `isMainActor: true`。

故事长度有三个预设：

| 预设 | 幕数 | 适合内容 |
| --- | ---: | --- |
| 短故事 | 3 幕 | 一次小互动、一个轻任务、课堂快速演示。 |
| 标准 | 5 幕 | 完整开端、问题、尝试、解决、结尾。 |
| 长故事 | 8 幕 | 活动剧情、多地点冒险、多角色合作。 |

### 预览区

预览区用于检查故事整体效果。建议每次 AI 生成后先看预览，再进入高级编辑。

重点检查：

- 标题是否清楚。
- 开头是否快速进入事件。
- 每一幕是否推动剧情。
- 互动任务是否自然嵌入故事。
- 宠物对白是否像儿童向宠物角色，而不是成人说明书。
- 场景背景是否与剧情匹配。
- 结尾是否有情绪完成感。

### 高级区

高级区有两个模式：

- 可视化编辑：逐幕编辑背景、场景标签、粒子、对白和互动。
- JSON 文本：直接编辑完整故事 JSON，适合开发、批量修改或从外部工具粘贴。

可视化编辑更适合日常创作。JSON 文本适合修复结构、复制模板、批量替换角色 ID 或检查保存数据。

## 四、完整创作流程

### 第一步：先确定故事目标

开始写提示词前，先想清楚四个问题：

1. 谁是主角？例如“玩家的抱抱龙”“两只宠物伙伴”。
2. 发生了什么问题？例如“水晶不会唱歌了”“小飞船迷路了”。
3. 玩家要做什么互动？例如“轻拍鼓励”“喂食补充力气”“洗干净道具”“完成小游戏”。
4. 结尾给玩家什么感受？例如“学会合作”“照顾伙伴”“勇敢表达”。

故事编辑器最适合“一个小问题 + 两三次行动 + 一个温暖结尾”的结构。不要把一个故事写成百科介绍，也不要让每一幕都只是角色聊天。

### 第二步：选择演员

在“选择演员”区域添加宠物演员。建议：

- 1 只宠物：适合个人成长、日记、照顾主题。
- 2 只宠物：适合合作、误会、互相帮助。
- 3 只宠物：适合团队任务，但对白要短，避免画面拥挤。

给演员命名时可以使用角色名，而不一定使用宠物原名。例如把“幼崽 #ABCD”改成“糯糯队长”，故事会更有代入感。

### 第三步：写故事提示词

提示词不需要很长，但要包含“主题、地点、冲突、互动、风格”。推荐格式：

```text
请生成一个儿童向互动故事：主角在[地点]发现[问题]，需要和伙伴一起[行动]。
故事要有[情绪/寓意]，包含[互动类型]，结尾让玩家觉得[结果]。
风格温暖、短句、有画面感，不要说教。
```

示例：

```text
抱抱龙在星球上发现一颗会唱歌的水晶突然不响了。
它和伙伴要去森林、浴室和小舞台寻找原因，中途需要轻拍鼓励、喂食补充力气，并完成一次小游戏。
故事要表达“朋友难过时，先陪伴再想办法”，语言温暖，适合 6-10 岁孩子。
```

再短一点也可以：

```text
两只宠物在海边发现迷路的小云雀，一起照顾它、帮它找回天空桥。包含轻拍、喂食和一次小游戏，结尾温暖。
```

### 第四步：生成故事

选择短故事、标准或长故事后，点击“AI 生成故事”。生成过程中会把 AI 输出同步到 JSON 文本区域。如果 AI 不可用，编辑器会创建一个可编辑草稿，方便继续手动修改。

生成成功后，编辑器会自动：

1. 解析 AI 返回的 JSON。
2. 合并当前选择的演员信息。
3. 根据 `sceneTags` 自动匹配预设背景。
4. 切到预览区。
5. 尝试自动保存到 `stories/<storyId>.json`。

### 第五步：预览和调整

预览时按故事播放顺序检查。发现问题后回到“高级”的可视化编辑：

- 对白太长：拆成两条或删掉说明性句子。
- 剧情跳跃：加一条旁白承接原因。
- 互动突兀：把互动前后对白改成“为什么要互动”和“互动完成后发生什么”。
- 场景不对：点该幕的“背景”进入场景设置。
- 宠物位置不对：在对白开头添加舞台指示，例如“(左侧，开心)”。

### 第六步：试玩

点击“试玩”会进入真实故事播放器。试玩时重点确认：

- 第一屏是否能吸引玩家继续。
- 点击、喂养、洗澡、小游戏是否能正常推进。
- 对白气泡是否挡住角色或重要画面。
- 背景图加载是否正常。
- 背景音乐和粒子是否符合情绪。
- 结尾是否自然结束。

### 第七步：保存和分享

点击“保存”后，编辑器会写入：

- `stories/<storyId>.json`：故事完整内容。
- `stories/index.json`：故事列表索引。

保存成功后界面会显示类似：

```text
已保存到 stories/my_story.json，已写入故事索引，分享参数：?story=stories/my_story.json
```

这个保存提示里的 `?story=stories/my_story.json` 主要用于当前用户自己检查保存路径。分享给别人时，应使用故事列表的分享按钮生成完整链接，格式类似：

```text
MagicHaqi.html?storyFrom=xiaoming&story=my_story.json
```

内部实现会用 `storyFrom` 作为 KeepWork 用户名，并把 `story` 自动补成 `stories/my_story.json`，再读取 `//xiaoming/edunotes/store/MagicHaqi/stories/my_story.json`。如果没有 `storyFrom`，播放器只会从当前登录用户自己的 workspace 或项目相对路径读取。

## 五、如何制作和设置场景

每一幕都可以设置场景。场景由四类信息组成：

| 类型 | 字段 | 作用 |
| --- | --- | --- |
| 背景图或底色 | `background` | 决定画面主体。 |
| 场景标签 | `sceneTags` | 用来自动匹配预设场景。 |
| 粒子 | `particles` | 增加氛围，例如星光、泡泡、雪。 |
| 音乐 | `bgMusic` | 播放该幕背景音乐。 |

### 1. 在故事编辑器里设置场景

进入“高级” -> “可视化编辑”，在某一幕点击“背景”，会打开“场景设置”。场景设置包含：

- 图片：选择预设场景，或 VIP 用户 AI 生成背景图。
- 颜色：设置没有图片时的背景色，也作为加载占位色。
- 粒子：选择星光、雪花、细雨、薄雾、泡泡、花瓣、暖光。
- 音乐：选择当前支持的背景音乐。

点“应用”后，场景会写回当前幕。

### 2. 使用预设场景

预设场景来自 [pet-story/presets/scenes.json](../pet-story/presets/scenes.json)。每个预设包含：

```json
{
	"id": "forest_glade",
	"title": "森林空地",
	"imageUrl": "https://cdn.keepwork.com/maisi/magichaqi/scenes/forest_glade-xxx.webp",
	"color": "#bbf7d0",
	"tags": ["outdoor", "forest", "land", "spring"],
	"particles": ["petals", "sparkle"],
	"bgMusic": "forest"
}
```

故事 AI 会给每幕生成 `sceneTags`，编辑器再根据标签自动匹配最接近的预设。比如：

- `forest, outdoor, spring` 会更容易匹配森林空地、阳光溪畔。
- `bathroom, indoor` 会更容易匹配泡泡浴室。
- `school, playground, land` 会更容易匹配学校操场。
- `seaside, ocean, sand` 会更容易匹配海边沙滩。

### 3. 常用场景标签

当前故事场景支持的常见标签包括：

```text
indoor, outdoor, land, sky, ocean, playground, bathroom, living room,
shop, school, spring, winter, seaside, haqi, townhall, forest, sand,
hospital, night, mountain, spaceship, castle, farm, park, zoo,
underwater, jungle, candy
```

可以在提示词里直接写这些标签，或写中文场景名。系统会识别部分中文别名，例如“森林”“浴室”“海边”“学校”“农场”“糖果”“飞船”。

### 4. 粒子效果选择

| 粒子 ID | 中文效果 | 适合场景 |
| --- | --- | --- |
| `sparkle` | 星光 | 魔法、水晶、夜晚、舞台。 |
| `snow` | 雪花 | 冬天、雪山、冰湖。 |
| `rain` | 细雨 | 雨天、森林、情绪低落。 |
| `mist` | 薄雾 | 海边、山地、神秘入口。 |
| `bubbles` | 泡泡 | 浴室、水下、海洋。 |
| `petals` | 花瓣 | 春天、花园、公园。 |
| `embers` | 暖光 | 火山、篝火、温暖结尾。 |

建议每幕最多 1 到 2 种粒子。粒子过多会影响画面清爽度。

### 5. AI 生成自定义背景

在“场景设置”里切到“图片” -> “AI 生成”，VIP 用户可以输入场景描述和参考图 URL 生成背景。生成后会插入当前故事作为自定义场景。

适合故事背景的提示词写法：

```text
明亮儿童向的哈奇星球森林空地，春天，浅绿色草地和彩色小花，远处有圆润树冠和柔和天空。
画面中央和下方保留宠物站立空间，不要文字，不要人物，不要黑边，适合移动端互动故事背景。
```

再比如：

```text
温暖的星球小医院室内背景，柔和薄荷绿和白色，圆角家具，窗外有阳光，画面中央留空给宠物角色站立。
儿童向、干净、明亮，不要文字，不要真实人物，不要复杂海报构图。
```

生成故事场景图时要特别注意：

- 背景不能有大段文字。
- 中央和底部要留角色空间。
- 不要让主体占满画面，否则宠物会挡住主体。
- 不要过暗、过写实或恐怖。
- 风格要和 MagicHaqi 的儿童向明亮视觉一致。

## 六、如何写好故事提示词

### 1. 推荐公式

```text
角色 + 地点 + 问题 + 行动 + 互动 + 情绪/寓意 + 风格约束
```

完整模板：

```text
请生成一个 MagicHaqi 儿童互动故事。
角色：[主角和伙伴关系]。
地点：[主要场景，例如森林、浴室、海边、飞船、学校]。
问题：[需要解决的小冲突]。
行动：[角色如何尝试解决]。
互动：包含 [tap/feed/bath/minigame 中的 2-3 个]，小游戏使用 [小游戏名或主题]。
情绪：开头有一点担心，中间互相鼓励，结尾温暖。
要求：中文短句，有画面感，每一幕都推动剧情，不要说教，不要长篇说明。
```

### 2. 好提示词示例

```text
请生成一个 5 幕儿童互动故事。两只宠物在冬日天空桥发现一盏星灯变暗了，大家需要去森林找风、去浴室洗净灯罩、再到小舞台点亮星灯。
包含轻拍鼓励、洗澡清洁和一次小游戏。故事表达“遇到问题先一起试一试”，对白短，画面温暖，不要说教。
```

```text
主角是一只胆小但善良的抱抱龙。它在学校操场听到新同学不敢上台表演，于是陪它练习。
故事包含轻拍鼓励、喂食补充力气和舞台演出结尾。语气适合小学生，短句，有一点幽默，结尾让玩家感到被鼓励。
```

```text
做一个 3 幕短故事：宠物在海边捡到一个不会发光的贝壳。它先清洗贝壳，再轻拍唤醒贝壳里的小星星，最后把贝壳挂回潮汐树。
场景用海边、浴室、夜晚星光。互动包含 bath 和 tap，结尾温柔。
```

### 3. 不推荐的提示词

```text
写一个好玩的故事。
```

问题：太空泛，AI 不知道主题、场景和互动方式。

```text
写一个很长很复杂的史诗冒险，有很多角色和很多支线。
```

问题：故事编辑器是移动端轻量互动故事，不适合复杂长篇和大量分支。

```text
介绍一下保护环境的重要性，越详细越好。
```

问题：会生成说教文本，不像可玩的宠物故事。更好的写法是“宠物发现河边有垃圾，和伙伴一起清理，让小鱼回来”。

### 4. 指定互动类型

故事编辑器当前支持四种互动：

| 类型 | JSON type | 用途 |
| --- | --- | --- |
| 轻拍互动 | `tap` | 鼓励、安抚、唤醒、约定。 |
| 喂养 | `feed` | 补充体力、分享食物、照顾伙伴。 |
| 洗澡/清洁 | `bath` | 洗干净宠物、道具、场景。 |
| 小游戏 | `minigame` | 关键挑战、训练、解谜或行动高潮。 |

提示词里可以明确写：

```text
互动必须包含一次 tap、一次 feed，最多一次 minigame。
```

小游戏建议只放一个。小游戏是强互动，太多会打断故事节奏。

### 5. 指定场景和标签

如果希望 AI 自动匹配到更准确的背景，可以在提示词里写场景标签：

```text
第 1 幕使用 forest/outdoor/spring，第 2 幕使用 bathroom/indoor，第 3 幕使用 sky/winter。
```

也可以用自然语言：

```text
故事依次经过春天森林、泡泡浴室、冬日天空桥。
```

### 6. 控制对白和舞台位置

对白可以用括号写舞台指示，播放时括号内容不会作为对白显示，只用来控制角色位置和状态。

常用写法：

```text
(左侧，开心)我找到第一颗星星了！
(右侧，伤心)可是它好像不亮了。
(中间，开心)我们一起试试吧。
(近处)把灯罩擦干净一点。
(远处，睡觉)呼呼，云朵还没醒。
```

支持的关键词包括：左、右、中间、中央、近处、远处、开心、伤心、睡觉，以及英文 `happy`、`sad`、`sleep`。

## 七、高级编辑和 JSON 结构

普通创作不需要手写 JSON，但理解结构有助于修复故事。

### 1. 最小故事结构

```json
{
	"id": "story_crystal_song",
	"title": "会唱歌的水晶",
	"version": 1,
	"selectionPrompt": "选择一位主角进入故事。",
	"actors": [],
	"startSceneId": "scene_1",
	"scenes": [],
	"ending": {
		"subtitle": "故事完成。",
		"text": "新的冒险已经准备好分享。"
	}
}
```

### 2. 单幕结构

```json
{
	"id": "scene_1",
	"sceneTags": ["forest", "outdoor", "spring"],
	"background": {
		"type": "image",
		"color": "#bbf7d0",
		"imageUrl": "https://cdn.keepwork.com/maisi/magichaqi/scenes/forest_glade-xxx.webp",
		"presetId": "forest_glade",
		"title": "森林空地"
	},
	"particles": ["petals"],
	"bgMusic": "forest",
	"timeline": [
		{ "kind": "line", "actor": "$narrator", "text": "森林里传来很轻的铃声。" },
		{ "kind": "line", "actor": "actor_1", "text": "(左侧，开心)我听见它在叫我们。" },
		{ "kind": "activity", "type": "tap", "title": "轻拍树洞", "count": 3, "successText": "树洞里亮起一点星光。" }
	]
}
```

### 3. 演员字段

演员通常由编辑器自动生成，不建议手动改 `sourcePetId`。常见字段：

| 字段 | 说明 |
| --- | --- |
| `id` | 故事内部角色 ID，对白和互动会引用它。 |
| `name` | 显示名。 |
| `sourcePetId` | 来源宠物 ID，用来关联玩家宠物。 |
| `isMainActor` | 是否主角。 |
| `allowUserSelection` | 播放前是否允许玩家选择该角色。 |
| `petTemplate` | 宠物外观模板，包含 DNA、图片、traits 等。 |

对白中的 `actor` 可以使用：

- `$narrator`：旁白。
- `$selected`：玩家选择的主角。
- `actor_1`、`actor_2` 等：具体演员 ID。

### 4. 时间轴元素

时间轴只有两类元素：

```json
{ "kind": "line", "actor": "$narrator", "text": "旁白内容。" }
```

```json
{ "kind": "activity", "type": "tap", "title": "轻拍鼓励", "count": 3, "successText": "伙伴安心地点点头。" }
```

`line` 用于旁白和对白，`activity` 用于可玩互动。每幕建议 3 到 6 个时间轴元素，其中 2 到 4 条对白，0 到 2 个互动。

### 5. 小游戏活动

小游戏活动需要 `gameId`：

```json
{
	"kind": "activity",
	"type": "minigame",
	"title": "完成运水小游戏",
	"gameId": "canal_escape",
	"gameTitle": "宠物运河营救",
	"count": 1,
	"successText": "清水顺着小渠流回来了。"
}
```

常见小游戏 ID 包括：

```text
pet_tower_defense, pet_bath, pet_snake, bubble_pets, match_three_pets,
food_stack_match, zuma, food_hexcells, canal_escape, sokoban,
laser_maze, lightbot, flappy_pet, xiangqi, gomoku, matrix_hack
```

选择小游戏时要让玩法和剧情一致。例如运河营救适合水流、救援、疏通；塔防适合守护、免疫、抵御；洗澡适合清洁主题。

## 八、当资源不够时怎么办

故事编辑器可以直接用已有宠物和预设场景快速创作。但如果你发现“没有合适场景”“没有主题角色”“活动星球缺少道具”，就要使用 [docs/dev_guide.md](dev_guide.md) 里介绍的内容生产工具补资源。

### 1. 缺少背景场景

使用 [dev_tools/ScenePresetsGenerator.html](../dev_tools/ScenePresetsGenerator.html)。

适用情况：

- 故事需要新地点，例如糖果工坊、魔法课堂、火山洞口。
- 星球 field 也需要复用这张背景。
- 想让多个故事共享同一套背景。

生产流程：

1. 打开 Scene Presets Generator。
2. 新建场景，填写英文 `id`、中文标题、底色、tags、粒子和音乐。
3. 写背景图提示词，生成或上传图片。
4. 确认图片明亮、无文字、中央留空、适合宠物站立。
5. 保存到 `pet-story/presets/scenes.json`。
6. 回到故事编辑器，在场景设置中刷新/搜索该场景。

建议命名：

```text
candy_cloud_stage
haqi_magic_classroom
volcano_cave_gate
moon_forest_bridge
```

### 2. 缺少官方角色或活动宠物

使用 [dev_tools/FamousPetGenerator.html](../dev_tools/FamousPetGenerator.html)。

适用情况：

- 活动故事需要固定 NPC。
- 品牌或课程需要专属角色。
- 希望玩家能通过图鉴、邀请链接或活动获得这只宠物。

生产流程：

1. 新建宠物 ID 和中文名。
2. 设置 traits、稀有度、阶段和动作。
3. 编写宠物外观提示词。
4. 生成 4x4 sprite sheet。
5. 检查 16 格角色一致、透明背景、动作清楚。
6. 保存 `famous-pets/<petId>.json`。
7. 更新 [famous-pets/index.json](../famous-pets/index.json)。
8. 在故事或活动入口中引用该宠物。

### 3. 缺少主题星球或独立产品入口

使用 [dev_tools/FamousPlanetGenerator.html](../dev_tools/FamousPlanetGenerator.html)。

适用情况：

- 要把故事活动包装成一颗主题星球。
- 需要固定 URL 入口，例如 `MagicHaqi.html?home_planet=haqi`。
- 想让故事、星球 field、商店商品形成同一主题。

生产流程：

1. 规划星球 ID、中文名、简介和 7 个 field。
2. 先用 Scene Presets Generator 准备场景背景。
3. 在 Famous Planet Generator 中创建星球。
4. 给 7 个 field 绑定场景、地形类型、粒子和音乐。
5. 如需专属商店，填写 `shopItemUrl`。
6. 保存到 [famous-planets/_planet_index.json](../famous-planets/_planet_index.json)。

### 4. 缺少故事道具或主题家具

使用 [dev_tools/ShopItemGenerator.html](../dev_tools/ShopItemGenerator.html)。

适用情况：

- 故事里提到“星灯”“水晶”“糖云小屋”，游戏里也希望能购买或摆放。
- 星球活动需要专属家具、食物、房屋。
- 想把故事奖励做成背包物品。

生产流程：

1. 选择默认商店或星球专属商店文件。
2. 新建物品，填写 ID、中文名、emoji、价格、类型。
3. 设置可摆放区域 `fields` 或房间 `rooms`。
4. 生成 SVG 或 WebP 视觉资源。
5. 保存商店 JSON。
6. 如果是星球专属商品，在星球配置里挂上 `shopItemUrl`。

### 5. 资源生产推荐顺序

如果要做一个完整主题活动，推荐顺序是：

1. 先写故事大纲和 3 到 5 个关键地点。
2. 用 Scene Presets Generator 创建这些地点。
3. 用 Famous Pet Generator 创建活动角色。
4. 用 Shop Item Generator 创建关键道具。
5. 如需独立入口，用 Famous Planet Generator 创建主题星球。
6. 回到故事编辑器生成故事，选择新场景和新角色。
7. 试玩、保存、分享。

## 九、故事质量检查表

保存和发布前建议逐项检查：

### 剧情

- 标题能概括故事。
- 第一幕能说明角色和问题。
- 每幕都推动剧情，没有空转聊天。
- 至少有 2 个互动，其中至少 1 个是轻量互动。
- 结尾有情绪完成感或小寓意。

### 文本

- 中文短句，适合儿童阅读。
- 对白像角色说话，不像系统说明。
- 每条对白不要太长。
- 不出现不适合儿童的恐怖、暴力、仇恨或成人内容。
- 不使用未授权的受保护角色或品牌形象。

### 场景

- 每幕场景与剧情一致。
- 背景图能正常加载。
- 宠物不会遮住关键主体。
- 粒子不超过 1 到 2 种。
- 音乐不突兀。

### 互动

- `tap/feed/bath/minigame` 类型正确。
- `count` 不要过大，通常 1 到 5 比较合适。
- `successText` 和剧情结果有关。
- 小游戏最多 1 个，除非是长故事活动。
- 互动前后都有剧情承接。

### 数据

- JSON 能正常保存。
- `actors` 没有丢失 `sourcePetId`。
- `startSceneId` 指向存在的场景。
- 每个 `scene.id` 唯一。
- 对白 `actor` 引用的是 `$narrator`、`$selected` 或真实演员 ID。
- 分享链接能打开。

## 十、常见问题

### 1. AI 生成失败怎么办

先检查登录和 SDK 状态。如果 AI 暂时不可用，编辑器会创建一个可编辑草稿。可以在高级区直接修改草稿，或稍后重新生成。

### 2. 场景匹配不准确怎么办

进入该幕“背景”手动选择预设。也可以在高级区修改 `sceneTags`，例如把 `forest` 改成 `bathroom, indoor` 后再选背景。

### 3. 背景列表里没有想要的地点怎么办

使用 Scene Presets Generator 创建新场景，保存到 [pet-story/presets/scenes.json](../pet-story/presets/scenes.json)。故事编辑器读取的预设都来自这个文件。

### 4. 角色不对或主角不对怎么办

回到“创作”区重新选择演员，或在高级 JSON 中检查 `actors` 和 `isMainActor`。不要随意删除 `sourcePetId`，否则故事播放时可能找不到对应宠物外观。

### 5. 播放时对白位置不好怎么办

在对白开头添加舞台指示，例如：

```text
(左侧，开心)我来试试看。
(右侧)我帮你拿着星灯。
(近处，伤心)它好像真的累了。
```

### 6. 互动无法推进怎么办

检查活动类型是否只使用 `tap`、`feed`、`bath`、`minigame`。旧字段如 `comfort`、`clean`、`play` 可能会被兼容转换，但正式故事建议使用当前四种标准类型。

### 7. 分享链接打不开怎么办

确认故事已保存，并且 `stories/index.json` 已更新。当前用户自己测试 workspace 故事时，可以使用：

```text
?story=stories/<storyId>.json
```

但分享给别人时必须包含用户名参数，推荐使用故事列表里的分享按钮生成：

```text
?storyFrom=<username>&story=<storyId>.json
```

这里的 `<username>` 是故事作者的 KeepWork 用户名。播放器会根据它去读取作者 workspace 下的 `stories/<storyId>.json`；如果少了 `storyFrom`，对方打开时会去找自己账号里的同名故事，自然就找不到。

### 8. 自定义 AI 背景没有长期保存到预设怎么办

故事编辑器里的 AI 生成背景会应用到当前故事，但如果要长期复用，应使用 Scene Presets Generator 正式保存到 [pet-story/presets/scenes.json](../pet-story/presets/scenes.json)，并使用稳定 CDN URL。

## 十一、推荐工作流范例

### 范例：制作“会唱歌的水晶”5 幕故事

1. 目标：表达“朋友难过时，先陪伴再想办法”。
2. 演员：选择当前宠物作为主角，再添加一只伙伴宠物。
3. 提示词：

```text
两只宠物在森林里发现一颗会唱歌的水晶突然不响了。
它们先轻拍鼓励水晶，再去厨房分享点心补充力气，最后到海天舞台完成一次小游戏让水晶重新唱歌。
故事 5 幕，温暖、有画面感，包含 tap、feed 和最多一次 minigame，结尾表达“陪伴能让声音回来”。
```

4. 生成：选择“标准”，点击 AI 生成故事。
5. 场景：检查森林、客厅/厨房、舞台是否匹配；不匹配就手动换背景。
6. 高级编辑：把过长对白拆短，给主角对白加“(左侧，开心)”等舞台指示。
7. 试玩：确认互动能推进，小游戏完成后有剧情反馈。
8. 保存分享：保存后复制分享链接。

### 范例：资源不够时补场景

如果现有预设里没有“水晶舞台”：

1. 打开 Scene Presets Generator。
2. 新建 `crystal_song_stage`。
3. tags 填 `outdoor, stage, sparkle, night`，如果 `stage` 不是常用标签，也可以补 `playground` 或 `haqi` 方便搜索。
4. prompt 写：

```text
明亮儿童向的星光水晶小舞台，中央有一颗温柔发光的水晶，周围是圆润云朵和彩色小旗。
画面下方留出宠物站立空间，不要文字，不要人物，不要黑边，适合移动端互动故事背景。
```

5. 生成并上传 CDN。
6. 保存到 `pet-story/presets/scenes.json`。
7. 回故事编辑器，在场景设置里搜索 `crystal` 或“水晶”。

## 十二、维护建议

- 常用场景优先做成预设，不要只放在单个故事里。
- 活动角色优先做成 famous pet，方便图鉴、邀请和多故事复用。
- 故事提示词里明确互动类型和场景顺序，生成质量会明显更稳定。
- 每次发布前至少完整试玩一次。
- 大型活动要把故事、星球、商店、宠物一起验证，因为它们会互相引用。
- 正式内容尽量使用 CDN 图片，不要长期依赖 `data:image/...`。
- 已上线故事的 `id` 和分享路径不要随意改；需要改版时可以保存为新故事。

故事编辑器最好的使用方式，是让 AI 先给出可玩的第一稿，再由创作者把“情绪、节奏、场景和互动”调到位。先让故事能玩，再让故事好玩。
