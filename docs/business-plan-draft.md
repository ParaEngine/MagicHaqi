# 蛋蛋星球 MagicHaqi 商业计划书草案

日期：2026-05-21

## 1. 项目摘要

蛋蛋星球 MagicHaqi 是一款移动端优先的轻量 H5 虚拟宠物养成游戏，继承“魔法哈奇 / 哈奇小镇”的情感资产，并用 AI 生成宠物、DNA 遗传、星球经营、宠物长期记忆和数字人语音对话，把传统宠物养成升级为“可持续陪伴 + 可分享内容 + 可召回老用户”的新产品。

短期目标不是先做一个重度 3D MMO，而是用低门槛 Web 游戏验证三件事：

1. 老哈奇用户是否愿意被抱抱龙、宠物伙伴和童年记忆召回。
2. 7-14 岁新用户是否能被 AI 专属宠物、互动绘本和社交分享吸引。
3. H5 是否能成为魔法哈奇手机 App、小程序、TapTap 版本和 KeepWork 账号体系的高效获客入口。

推荐路径：先做“抱抱龙召回 + AI 宠物互动绘本分享 + 授权经典场景互动孵化”的 H5 增长闭环，再把验证成功的内容迁移到微信小程序、TapTap 和手机 App。5 个月内重点完成可留存、可分享、可付费、可合作招商的最小商业闭环。

## 2. 背景与机会

魔法哈奇 / 哈奇小镇是 2009 年团队开发的 3D 产品，曾拥有约 500 万注册用户。早期用户当年多为 7-14 岁，如今已经进入 20 岁以上阶段。这批用户的价值不只是“老账号”，而是对抱抱龙、哈奇小镇、宠物伙伴和童年社交体验的情感记忆。

当前市场同时出现了两个新机会：

- 怀旧召回：赛尔号、摩尔庄园等 IP 证明，早年儿童互联网产品在成年后仍有情绪召回价值，但重度 App 和 MMO 开发成本高，验证周期长。
- AI 陪伴内容：AI 图像生成、AI 角色聊天和数字人语音让“每个用户拥有独一无二宠物”成为可规模化实现的体验。
- 授权内容合作：经典复古儿童游戏、动画角色、教育品牌或活动 IP 可以不必先做完整联名游戏，而是以“场景故事 + 限定宠物”的轻量形式进入 MagicHaqi，快速验证转化与商业价值。
- IP 宠物化产品：线下儿童动漫 IP、快消品牌、亲子品牌和实体门店过去常依赖玩具、贴纸、卡牌等低互动赠品连接用户，成本高、玩法单一、难以沉淀长期关系。MagicHaqi 可以把一个虚拟 IP 变成“可养成、会讲故事、可互动分享”的专属在线宠物，把一次性联名赠品升级为持续复购、会员权益和线下到线上的长期陪伴入口。
- 中小 IP 零成本产品化：大量中小 IP、原创形象、课程角色和区域品牌缺少足够流量与研发预算，无法独立开发 App 或小游戏。MagicHaqi 可以提供现成的星球、宠物、家具、互动故事和分享系统，让他们用极低成本发行自己的专属养成产品，在小圈层内传播并提升付费能力和影响力。

还有一个更大的潜在机会：同代儿童网游用户高度重叠。哈奇早期用户很多来自摩尔庄园、赛尔号等同期大 IP 的产品内引流，这些产品曾覆盖上亿注册级别的儿童互联网用户。对今天的 20 岁以上怀旧人群来说，哈奇、摩尔庄园、赛尔号等记忆并不是割裂的单点，而是一整段“同一时期、同一批孩子、同一套网页游戏生活方式”的共同记忆。如果 MagicHaqi 的抱抱龙故事先跑通，就可能激发用户自然讨论、再创作和期待：能不能也用类似方式重温同期其他儿童游戏里的故事、场景和宠物。

这意味着 famous-pets 不只是一个稀有宠物系统，还可能成为“同代 IP 记忆网络”的入口。它的正确边界是：自有哈奇内容优先，外部 IP 必须走授权合作；用户侧可以支持原创致敬和相似情绪模板，但不能鼓励上传或复刻未授权截图、角色名、商标和原始素材。

MagicHaqi 当前形态适合承担前哨产品：无需下载安装即可体验，用户先孵化一只宠物，再通过成长、互动、分享和哈奇岛入口被引导下载魔法哈奇手机 App。

从更大的平台角度看，MagicHaqi 也可以成为“IP 星球发行工具”：IP 厂商不只是接入一只联名宠物，而是可以用星球创作工具定义自己的主题星球、场景、宠物形象、家具、剧情模板和活动入口。对大型品牌来说，它可以成为线下活动、套餐赠品、会员权益和线上互动之间的桥梁；对中小 IP 来说，它可以成为不需要自建技术团队的轻量在线养成产品。如果足够多的 IP 用同一套工具发行自己的星球，就可能形成多品牌、多角色、多故事之间的私域病毒传播网络：用户因一个 IP 进入，因宠物成长、AI 故事、好友互访和跨品牌互动继续停留，并把新的故事卡片带回各自的小圈层。

## 3. 当前产品基础

当前 MagicHaqi 已具备以下商业化前置能力：

| 能力 | 当前价值 | 商业含义 |
| --- | --- | --- |
| H5 直接打开 | 低门槛体验 | 适合社媒传播、老用户召回、广告落地页 |
| KeepWork 登录与存档 | 账号资产沉淀 | 可转化到 App、小程序、课程和会员体系 |
| AI 生成宠物立绘 | 每只宠物独特 | 支持付费生成、稀有外观、分享传播 |
| DNA 与繁殖 | 可收集、可继承 | 支持长期养成、交易/展示、活动限定基因 |
| famous-pets 稀有宠物 | 可配置授权宠物 | 支持联名宠物、活动宠物、经典角色回归、付费孵化 |
| 互动 MV 式孵化 | 低交互故事体验 | 支持用授权截图/场景/角色做轻量内容活动 |
| AI 故事创作器早期版本 | 可选择真实宠物演员，生成结构化互动故事 JSON | 支持低成本生产宠物绘本、互动 MV 脚本、授权活动样片和 UGC 创作入口 |
| 星球定居链接 | 可用 `home_planet` 参数指定初始主题星球 | 支持按老用户、课程、节日、联名品牌等人群创建差异化世界入口 |
| 星球经营 | 中期目标 | 提升留存，承接资源循环和装饰付费 |
| 宠物长期记忆 | 情感绑定 | 支持 AI 陪伴会员与复访 |
| 数字人语音入口 | 高级体验 | 可作为 VIP 功能或 App 下载理由 |
| 哈奇岛 / App 入口 | 召回承接 | H5 成为 App 获客和唤醒页 |

Story Maker 的早期形态已经具备商业验证价值：它不是单纯文本生成，而是把宠物演员、场景字幕、对白、喂养/清洁/轻拍/安抚/小游戏等真实游戏动作，组织成可预览、可试玩、可保存的结构化故事。短期内可先作为运营内部工具和种子用户创作工具，用于快速生成“抱抱龙回忆故事”“孩子的宠物绘本”“授权经典场景互动 MV”样片，再逐步开放给普通用户。

MagicHaqi 还支持通过链接参数指定玩家的初始母星模板，例如 `MagicHaqi.html?home_planet=famous-planet/haqi`。这个“星球定居”能力可以为不同用户群体创建不同主题世界：老哈奇用户进入哈奇怀旧星球，亲子用户进入绘本或课程主题星球，合作品牌用户进入联名活动星球。它的商业价值在于把同一套宠物养成底层系统包装成多个可投放、可运营、可招商的垂直入口，同时仍然沉淀到统一账号和个人存档中。

进一步看，星球创作工具可以从内部运营工具升级为 IP 厂商的自助发行后台。合作方可以自己配置主题星球的视觉氛围、场景、宠物/IP 角色、家具、活动任务、剧情模板和分享卡文案。大型 IP 或实体品牌可以把星球作为线上会员中心、线下核销入口和活动复购入口；中小 IP 则可以把星球作为自己的轻量产品首页，不必承担完整游戏研发、服务器和长期运营成本。

产品尚需补强的核心短板：新手引导、抱抱龙/哈奇 IP 首屏信号、分享链路、商业化包装、老用户召回验证数据、授权内容管理、儿童与家长合规设计，以及 Story Maker 生成内容的审核、模板化和分享转化。

## 4. 目标用户市场

### 4.1 老哈奇用户：20 岁以上怀旧人群

这是前 5 个月最值得优先验证的人群。

他们的特征：

- 记得哈奇小镇、抱抱龙、宠物伙伴、童年网页游戏社交。
- 有一定消费能力，但不一定愿意立刻下载新 App。
- 更容易被“找回当年的抱抱龙”“给童年宠物写一封信”“孵化你的新抱抱龙”触发。
- 愿意在小红书、B站、抖音、微信群、QQ 群中分享怀旧内容。

对他们的产品承诺应该是：不是复制老游戏，而是让童年宠物以 AI 方式重新陪伴你，并提供通往新版魔法哈奇 App 的入口。

### 4.2 7-14 岁新用户

这是中长期增长人群，需要更谨慎设计。

他们的特征：

- 对“专属宠物、孵蛋、装扮、小游戏、语音互动、绘本分享”敏感。
- 获取渠道更多来自微信小程序、短视频、TapTap、家长推荐和校园场景。
- 需要更强的新手引导、更明确的安全边界、更轻的文本阅读压力。

对他们的产品承诺应该是：养一只会长大、会记得你、能一起创造故事的神奇宠物。

### 4.3 家长与教育用户

家长不是第一消费触发点，但会影响 7-14 岁用户能否长期留存和付费。

潜在价值：

- AI 宠物互动绘本可包装为“孩子自己的故事书”。
- KeepWork 可承接编程、AI 创作、数字素养课程。
- 家长更容易为安全、创作、学习和陪伴价值付费，而不只是为游戏道具付费。

### 4.4 IP 品牌与实体合作方

这是 MagicHaqi 除 C 端用户之外最值得验证的 B 端客户。

他们的特征：

- 拥有儿童动漫角色、吉祥物、课程角色、门店会员、套餐活动或区域小 IP。
- 过去主要通过线下玩具、贴纸、卡牌、立牌或一次性 H5 活动触达用户，成本高、留存弱、复购链路短。
- 希望把角色做成持续可互动的在线资产，但缺少独立开发游戏和长期运营的预算。
- 中小 IP 尤其需要“低成本上线、在私域小圈层传播、能带来付费或影响力增长”的工具。

对他们的产品承诺应该是：不用从零开发游戏，也能拥有一个以自身 IP 为核心的宠物养成星球；用户通过养宠、故事互动、好友分享和线下权益兑换，持续和品牌发生关系。

## 5. 可选迭代方向挑战

本节刻意把几个看似合理的方向放到一起挑战，避免过早押注单一路线。

### 方向 A：老用户召回优先

核心假设：抱抱龙和哈奇小镇记忆足以带来低成本召回。H5 只要快速让用户孵化或认领一只宠物，就能引导下载 App 或加入社群。

优势：

- 情感钩子强，营销文案清晰。
- 20 岁以上用户有付费能力，能接受会员、纪念皮肤、AI 生成内容。
- 可通过老账号、社群、短视频怀旧内容快速测试。

挑战：

- 如果老 IP 元素不够明显，用户可能只觉得这是一个普通 AI 宠物 H5。
- 怀旧用户可能只回来打卡一次，长期留存未必高。
- 过度怀旧会限制新用户理解。

结论：必须优先做，但不能只做怀旧。召回入口应服务于更大的 AI 宠物养成闭环。

### 方向 B：7-14 岁新用户优先

核心假设：新一代用户会被 AI 专属宠物、装扮、互动绘本和小程序传播吸引。

优势：

- 市场空间更长期。
- 更适合微信小程序、TapTap、校园和 KeepWork 教育场景。
- 可持续扩展宠物、装扮、小游戏、绘本和亲子创作。

挑战：

- 冷启动成本高，没有老 IP 情绪加成。
- 儿童内容需要更高安全、合规、家长控制和内容审核投入。
- 如果前期玩法深度不足，用户流失会很快。

结论：作为第二阶段目标，不适合在前 1-2 个月成为唯一主线。先用老用户召回验证核心宠物体验，再面向新用户重包装。

### 方向 C：AI 宠物互动绘本优先

核心假设：用户愿意把宠物互动自动生成绘本、短图文或视频卡片分享到社媒，形成自传播。

当前 Story Maker 已经提供了这个方向的早期产品基础：用户可以选择自己已有的宠物作为演员，用一句提示词生成包含多幕场景、对白、互动动作和小游戏节点的故事 JSON，并在移动端预览、微调、试玩和保存。这使“互动绘本”不必从零开始开发，而可以先从结构化故事和可分享故事卡片演进。

优势：

- 与 AI 能力强相关，差异化明显。
- 比单纯截图更适合小红书、朋友圈、抖音图文、B站动态。
- 可自然连接家长市场：孩子和宠物共同创作故事。

挑战：

- 生成质量、速度和成本需要控制。
- 分享内容必须足够好看、好懂、带品牌识别。
- 如果没有玩法留存，分享只会带来浅层流量。

结论：非常适合作为增长引擎，但要建立在“孵化宠物 + 互动事件 + 一键生成分享卡”的基础上，而不是独立做成绘本工具。

### 方向 D：授权经典场景互动孵化优先

核心假设：用户不仅想“看见经典角色”，还愿意在熟悉的复古儿童游戏场景里完成一个很短的互动故事，并把故事最终孵化出的限定宠物带回自己的星球。这个玩法可以承接 famous-pets 机制：合作方提供授权截图、背景、角色和宠物配置，MagicHaqi 提供轻互动故事、成长阶段展示和星球养成承接。

Story Maker 可以成为这个方向的低成本生产后台：运营或合作方先用自然语言生成 3-5 幕互动故事，再在高级 JSON 和时间轴编辑器里调整对白、背景图 URL、动作次数和小游戏节点，最后交给 H5 故事播放器和 famous-pets 孵化流程承接。这会显著降低联名活动从商务创意到可试玩样片的时间。

体验形态：

1. 用户进入一个授权经典场景，例如复古儿童游戏关卡、童年场景截图或品牌故事背景。
2. 场景像互动 MV 一样推进，用户只需在每个背景里完成非常简单的点击互动，例如喂养、清洁、安抚、寻找道具或点击足够次数。
3. 每完成一段，宠物从阶段 1 进入阶段 2、阶段 3、阶段 4，画面和故事字幕随之推进。
4. 故事看完后，对应 famous-pet 被孵化成一只限定宠物，落入用户自己的蛋蛋星球。
5. 用户可以在宠物图鉴的“稀有宠物”中查看它的相册，并分享“我从某个经典场景带回了这只宠物”。

优势：

- 比完整联名游戏轻得多：一组授权背景、角色素材、宠物 4×4 图和短脚本就能上线一次活动。
- 互动门槛极低，适合 H5、微信、小红书和短视频导流用户。
- 能把合作 IP 的怀旧情绪转化为 MagicHaqi 的长期宠物资产，而不是一次性浏览。
- 有机会触发同代 IP 记忆外溢：哈奇用户往往也玩过摩尔庄园、赛尔号等同期产品，一旦抱抱龙故事火了，用户会自然把这种“看故事、点互动、孵化宠物”的形式投射到更多童年游戏记忆上。
- 可形成内容合作招商飞轮：先用自有哈奇故事证明完成率和分享率，再用数据去谈同期儿童游戏、动画、教育品牌或亲子 IP 的授权合作。
- 适合商业合作：品牌方买的是一次“可孵化、可收藏、可分享”的互动内容活动。

挑战：

- 必须使用已授权素材，不能把受版权保护的截图、角色或名字当作普通素材库使用。
- 用户可能会自发上传或要求复刻摩尔庄园、赛尔号等外部 IP 内容，需要提前设计审核、举报和下架机制。
- 如果互动太弱，用户会觉得只是换皮广告；如果流程太长，又会降低完成率。
- 合作方素材风格和 MagicHaqi 宠物体系需要统一包装，否则会破坏世界观。

结论：这是 famous-pets 最有商业想象力的方向。它不应替代 AI 宠物主线，而应作为“联名活动 + 怀旧召回 + 限定孵化”的增长插件，优先用于月度活动和商务合作验证。

### 方向 E：IP 星球发行工具优先

核心假设：很多 IP 方并不需要完整定制游戏，而需要一个能快速把角色变成“可养成宠物 + 可互动故事 + 可分享星球”的轻量产品。MagicHaqi 可以把星球创作工具产品化，让 IP 方自行发行主题星球，定义自己的场景、宠物、家具、剧情模板、活动任务和分享物料。

典型场景：

1. 大型品牌联名：例如快餐、亲子乐园、文具、零食或线下门店与儿童动漫 IP 合作，用户购买套餐后不只得到一次性玩具，还能扫码领取一只线上 IP 宠物，持续喂养、解锁故事、兑换线下权益。
2. 中小 IP 产品化：原创漫画角色、课程吉祥物、地方文旅形象、校园活动角色等，可以用极低成本拥有一个专属养成星球，通过粉丝群、公众号、社群和线下活动传播。
3. 多 IP 私域传播：不同 IP 的星球可以共享底层账号、宠物养成、AI 故事和好友互访机制，用户从一个 IP 进入后，可能因为跨星球互动、故事串联和活动互访接触更多品牌。

优势：

- 对品牌方来说，比线下玩具更低边际成本、更可追踪、更能沉淀用户关系和复购。
- 对中小 IP 来说，不需要自建研发团队，也能拥有可持续运营的在线产品。
- 对 MagicHaqi 来说，可以从单一游戏变成 IP 发行平台，获得服务费、模板费、活动分成、会员分成和联名权益收入。
- 星球、宠物、家具和故事模板天然适合批量复制，能形成多品牌内容供给。
- AI 故事可以让不同品牌、不同宠物之间产生低成本互动内容，增加传播话题。

挑战：

- 自助工具需要足够简单，否则 IP 方仍然依赖定制开发和人工配置。
- 多品牌共存需要统一质量标准、审核机制和儿童内容安全边界。
- 如果只做品牌露出而没有养成、故事和权益闭环，用户会把它当成普通广告 H5。
- 平台需要处理 IP 授权边界、素材归属、收益分成和下架机制。

结论：这是 MagicHaqi 从游戏产品升级为“IP 宠物化平台”的重要方向，但不宜在最早阶段抢走 C 端验证资源。建议先用自有哈奇星球和 1-2 个合作样板证明完成率、分享率、复购或权益核销，再逐步开放为可复制的星球发行后台。

### 方向 F：TapTap / App 商店游戏化优先

核心假设：把 MagicHaqi 快速包装成独立游戏，上 TapTap 和应用市场获取自然玩家。

优势：

- 平台用户天然愿意试游戏。
- 评论、预约和评分可沉淀公开口碑。
- 可为正式 App 版本做早期社区。

挑战：

- TapTap 用户对完成度要求高，当前 H5 的内容厚度和稳定性可能不足。
- 需要版本包、合规、隐私、未成年人保护和运营素材。
- 如果太早上线，差评会伤害品牌。

结论：不宜作为第一个月主战场。应在第 3-4 个月已有较完整新手、留存和分享闭环后，以测试版或预约页进入。

### 方向 G：重做完整 3D 魔法哈奇 App 优先

核心假设：老用户真正想要的是完整 3D 小镇体验，因此应直接集中资源做 App。

优势：

- 终局价值最大。
- 可承接社交、地图、宠物、装扮、小游戏、会员等完整商业化。
- 更符合“魔法哈奇”原始品牌想象。

挑战：

- 开发成本高，5 个月内难以完成高质量闭环。
- 在未验证召回效率前，直接重投入风险大。
- H5 已经可以更快验证宠物、分享和转化假设。

结论：作为中长期目标，但前 5 个月不应让重 3D App 吞噬 H5 验证速度。H5 应成为 App 的前置漏斗和玩法试验田。

## 6. 挑战后的最佳路径

最佳路径不是单选题，而是分阶段组合：

1. 第 1 阶段用“抱抱龙 / 哈奇小镇怀旧召回”获得第一批用户。
2. 第 2 阶段用“AI 专属宠物 + 互动绘本分享”扩大传播。
3. 第 3 阶段用“famous-pets 稀有宠物 + 授权经典场景互动孵化”验证内容合作和限定宠物付费。
4. 第 4 阶段用“星球定居链接 + 主题世界模板”把不同人群导入对应世界，提高活动转化和内容合作复用效率。
5. 第 5 阶段用“星球创作工具 + IP 厂商自助配置”验证品牌方是否愿意发行自己的宠物星球。
6. 第 6 阶段用“宠物记忆、星球经营、装饰收集”提高留存。
7. 第 7 阶段把高意愿用户导向“魔法哈奇手机 App / 微信小程序 / TapTap 测试”，并验证会员、AI 生成次数、语音互动、限定宠物、装扮和绘本导出。

一句话策略：用 H5 做低成本召回与分享增长，用 AI 宠物建立新体验，用星球定居链接承接分人群主题世界，用 famous-pets 承接授权内容合作，用星球创作工具把 IP 宠物化能力开放给品牌方和中小 IP，用 App 和小程序承接深度留存与商业化。

## 7. 5 个月开发与推广计划

### 第 1 个月：召回 MVP 与新手闭环

开发重点：

- 重构首屏，让“魔法哈奇 / 哈奇小镇 / 抱抱龙”成为第一眼信号。
- 新手流程改为 3 分钟内完成：打开页面 → 登录/游客体验 → 认领或孵化抱抱龙 → 第一次喂食/互动 → 获得分享卡。
- 增加老用户召回页：文案围绕“还记得你的抱抱龙吗？”“回到哈奇岛前，先孵化一只新伙伴”。
- 增加星球定居参数入口：支持 `home_planet=famous-planet/haqi` 这样的主题母星链接，用于老用户召回、课程活动和合作投放分流。
- 打磨 Story Maker 内部样片能力：先用现有故事创作器制作 3 条抱抱龙回忆故事，用于验证“对白 + 轻互动 + 分享卡”的内容吸引力。
- 增加轻量埋点：访问、开始孵化、完成孵化、首次互动、分享点击、App 下载点击、次日回访。
- 加入基础游客体验，降低登录门槛；需要保存和领取奖励时再引导 KeepWork 登录。

推广重点：

- 内部老用户渠道、QQ群、微信群、公众号、B站动态、小红书怀旧帖。
- 找 30-100 名老用户做可访谈测试。
- 目标不是大曝光，而是验证“怀旧钩子 + 孵化宠物”是否成立。

关键指标：

- 首次孵化完成率 >= 45%。
- 首次分享点击率 >= 15%。
- App 下载入口点击率 >= 8%。
- 次日回访率 >= 15%。

### 第 2 个月：互动绘本与社媒传播

开发重点：

- 上线“宠物互动绘本”最小版本：根据喂食、玩耍、聊天、星球事件生成 3-6 张图文卡。
- 将 Story Maker 的结构化故事转成可分享内容：支持从故事封面、分幕预览和宠物对白生成图文卡，并沉淀精选故事模板。
- 上线第一版 famous-pets 稀有宠物图鉴：支持未拥有隐藏、拥有后解锁相册、金币或孵化券孵化。
- 分享卡带清晰品牌、水印、二维码和“孵化你的抱抱龙”入口。
- 提供 3 类模板：怀旧回忆、今日陪伴、孩子的宠物故事。
- 宠物聊天和记忆做体验打磨：让宠物能提到自己的名字、性格、最近事件。
- 控制 AI 成本：免费用户每日有限次数，失败时有模板兜底。

推广重点：

- 小红书：童年回忆 + AI 宠物绘本。
- 抖音/视频号：30 秒“我找回了 2009 年的抱抱龙”。
- B站：开发日志、哈奇小镇回忆、AI 宠物演示。
- 公众号：面向家长和老玩家讲“童年宠物回来了”。

关键指标：

- 分享生成率 >= 25%。
- 分享带来的新访问占比 >= 20%。
- 7 日留存 >= 8%。
- 单次 AI 生成平均成本可控，免费额度不造成成本失控。

### 第 3 个月：留存系统与轻商业化

开发重点：

- 打磨日常养成：每日任务、连续陪伴、宠物状态提醒、星球设施目标。
- 增加抱抱龙/经典宠物伙伴系列外观或基因池，形成收集目标。
- 制作第一条“授权经典场景互动 MV”样例：用 3-5 个背景场景讲一个短故事，玩家通过点击喂养、清洁、安抚或整理道具推进剧情，故事结束后孵化一只 famous-pet 进入星球。
- 把 Story Maker 升级为运营可用的活动脚本工具：预置授权活动模板、素材审核字段、故事试玩链接和一键复制配置，方便商务演示和合作方确认。
- 建立 famous-pets 活动配置标准：`index.json` 管理稀有度、价格和图鉴展示，单只宠物 JSON 管理授权宠物形象、阶段和故事承接。
- 上线会员雏形：更多 AI 生成次数、高清分享图、语音对话、稀有孵化特效、专属装饰。
- 增加 App 下载动机：在 H5 中展示“去 App 进入哈奇岛 / 更完整 3D 小镇 / 更多宠物伙伴”。
- 完善家长与未成年人保护入口：时长提示、付费提示、内容安全说明。

推广重点：

- 开始做 App 预约或下载转化 A/B 测试。
- 与 KeepWork 用户体系结合，向已有教育/创作用户推送。
- 组织第一次线上活动：“晒出你的新抱抱龙”。
- 启动第一轮授权素材合作洽谈：复古儿童游戏、教育品牌、亲子内容或自有哈奇经典素材优先。

关键指标：

- 7 日留存 >= 12%。
- 会员入口点击率 >= 5%。
- 稀有宠物详情页到孵化点击率 >= 8%。
- 互动 MV 完成率 >= 50%。
- App 下载或预约转化率 >= 5%。
- 每用户日均 AI 成本低于预计商业化收入上限。

### 第 4 个月：微信小程序与 TapTap 测试

开发重点：

- 将 H5 核心流程适配微信小程序外壳或小程序 WebView 策略。
- 准备 TapTap 测试素材：游戏介绍、截图、视频、开发者日志、测试资格说明。
- 优化移动端性能、加载速度、离线回归和弱网体验。
- 增加社交任务：邀请好友孵蛋、互访星球、交换绘本祝福。
- 增加内容运营后台或配置表：活动宠物、famous-pets、互动 MV 场景、限定家具、节日模板、主题母星模板和 `home_planet` 投放链接。

推广重点：

- 微信生态做亲友传播和家长传播。
- TapTap 以“AI 生成专属宠物 + 哈奇小镇精神续作测试”定位进入。
- 寻找怀旧游戏、AI 应用、亲子创作类 KOL 小规模投放。

关键指标：

- 小程序访问到孵化完成率 >= 35%。
- TapTap 预约/关注转化率达到同类测试产品可接受水平。
- 邀请带来的新用户占比 >= 15%。
- 评论中对“AI 宠物独特性”和“哈奇怀旧感”的正反馈明显高于负反馈。

### 第 5 个月：商业闭环验证与路线决策

开发重点：

- 正式测试付费：月会员、AI 生成包、高清绘本导出、限定抱抱龙皮肤、语音陪伴。
- 正式测试联名/活动孵化：限定孵化券、活动通行证、品牌联名宠物、互动 MV 赞助位。
- 正式测试 Story Maker 增值能力：会员可获得更多故事生成次数、高清故事卡导出、精选模板、多人宠物演员和活动投稿资格。
- 梳理 H5 → App / 小程序 / TapTap 的用户分层：浅体验用户、怀旧用户、儿童用户、付费高意愿用户。
- 做数据看板：获客成本、激活率、留存、分享系数、App 转化、付费转化、AI 成本。
- 明确下一阶段是扩大 H5、小程序增长，还是投入更完整 App / 3D 小镇。

推广重点：

- 做一次更完整的“抱抱龙回归季”活动。
- 做一次“经典场景带回宠物”活动：看完一个复古互动故事，把限定宠物带回自己的星球。
- 联合老用户故事征集：我的哈奇小镇回忆、新抱抱龙绘本。
- 发布阶段性开发者报告，提升社区信任。

关键指标：

- 付费转化率达到早期验证线：活跃用户 1%-3%。
- 会员或生成包收入能覆盖 AI 生成与语音成本。
- H5 到 App / 小程序深度体验转化路径清晰。
- 确定下一阶段主投入方向。

## 8. 商业化模型

### 8.1 免费体验

免费用户应能完整体验核心快乐：孵化宠物、日常照料、基础装饰、生成低频分享卡、进入哈奇岛下载入口。免费体验不能太吝啬，否则传播会断。

### 8.2 会员订阅

适合包装为“哈奇伙伴会员”或“星球守护会员”。

可包含：

- 每日更多 AI 图像生成次数。
- 宠物语音 / 数字人对话。
- 高清互动绘本导出。
- 专属抱抱龙外观、稀有 DNA 特效。
- 更多宠物数量上限、更多房间/星球装饰槽位。

### 8.3 消耗型付费

适合低价、明确、非强迫：

- AI 立绘重生成包。
- 绘本导出包。
- AI 故事生成次数包或高清故事卡导出包。
- 限定孵化券。
- famous-pets 稀有宠物孵化券。
- 授权互动 MV 活动通行证。
- 节日主题装饰包。

### 8.4 授权内容合作

famous-pets 可以成为 MagicHaqi 的轻量联名商业模块。合作方不需要开发完整游戏，只需要提供已授权素材和角色设定，MagicHaqi 把它包装成一次“看故事、轻互动、孵化宠物、带回星球”的活动。

Story Maker 可以让联名合作更像“快速内容工厂”：商务前期先用合作方素材和角色设定生成可试玩样片，运营上线前再锁定审核后的背景、对白、动作节点和 famous-pet 奖励。这比传统定制小游戏更轻，也更容易按月度活动复制。

这个模块的增长逻辑不是单次联名，而是“同代 IP 记忆飞轮”：哈奇故事先验证怀旧召回和互动孵化形式；如果用户讨论和分享中自然提到摩尔庄园、赛尔号等同期大 IP，MagicHaqi 就可以把这种需求整理成合作提案，向外部版权方证明：不需要重做完整游戏，只需一条短互动故事和一只限定宠物，就能把老用户情绪转化为可留存的宠物资产。

可合作内容包括：

- 经典复古儿童游戏截图或场景背景。
- 合作方角色、宠物或吉祥物。
- 节日活动场景、品牌故事场景、教育课程场景。
- 自有哈奇经典内容回归，例如抱抱龙、宠物伙伴、哈奇岛主题。

可售卖权益包括：

- 联名 famous-pet 上架。
- 互动 MV 场景露出。
- Story Maker 联名故事模板和精选活动页露出。
- 限定孵化券或活动礼包。
- 分享卡品牌露出和二维码跳转。
- 线下/课程/会员权益兑换码。

合规边界：所有外部 IP 内容必须是明确授权、合作方提供或经过审核的原创致敬素材。平台可以鼓励用户创作“童年网页游戏风格”“复古小镇冒险”等情绪模板，但不能鼓励未授权复刻具体角色、截图、商标或宠物名称。

### 8.5 IP 星球发行与品牌私域合作

MagicHaqi 的更大 B 端商业化方向，是把“星球 + 宠物 + 家具 + AI 故事 + 互动任务 + 分享卡”包装成 IP 方可发行的在线养成产品。

对大型 IP 和实体品牌来说，这可以解决过去线下联名玩具的几个问题：

- 线下玩具成本高、库存重、生命周期短；线上宠物可以低边际成本持续运营。
- 传统赠品只有一次领取价值；线上宠物可以通过喂养、故事、任务、节日活动和线下权益兑换带来复购。
- 线下互动数据难沉淀；线上星球可以追踪领取、养成、分享、回访、兑换和付费。
- 品牌和动漫 IP 之间的合作不再只是“把角色印在玩具上”，而是可以共同设计一个有剧情、有成长、有会员权益的宠物陪伴产品。

对中小 IP 来说，这个模式更像“零成本专属产品”：他们不需要自建 App、服务器、游戏研发和内容工具，只需要提供角色设定、视觉素材和活动目标，就能获得一个可传播的专属星球。中小 IP 可以在自己的粉丝群、公众号、课程社群、线下活动和小红书内容中传播这个星球，用小圈层流量提升付费、曝光和粉丝黏性。

可收费方式包括：

- 星球发行服务费：为品牌或 IP 创建专属星球、宠物、家具和故事模板。
- 模板与素材包费用：提供节日活动、门店活动、课程任务、品牌剧情等标准模板。
- 活动运营费：按月维护联名活动、排行榜、兑换码、故事精选和分享页。
- 权益核销与复购分成：用户通过星球领取优惠券、套餐兑换、会员权益或线下奖励时分成。
- 虚拟商品分成：联名宠物皮肤、家具、故事生成、高清导出、限定孵化券等收入分成。
- 多 IP 联动活动费：多个品牌或角色共同参与主题活动，形成跨星球任务和故事串联。

这个方向的核心不是做一次广告 H5，而是帮助 IP 方拥有自己的“可持续在线宠物资产”。当大量 IP 都以星球形式接入后，平台有机会形成私域病毒传播：每个 IP 带来自己的粉丝小圈层，用户通过宠物成长、AI 故事、互访星球、跨品牌任务和分享卡片在圈层之间流动。

### 8.6 App 转化价值

H5 本身可商业化，但更大的价值是获客和转化：

- 老用户通过 H5 被唤醒，再下载魔法哈奇 App。
- 新用户通过分享进入，再在 App / 小程序中深度养成。
- KeepWork 账号沉淀后，可连接课程、创作工具、AI 学习产品。

### 8.7 品牌与内容价值

如果互动绘本跑通，MagicHaqi 不只是游戏，还可以成为用户生成内容平台：每个宠物都是一个角色，每段互动都能变成故事。这会带来更长线的 IP 内容资产。

Story Maker 让这件事更具体：宠物不只是被截图分享，而是可以成为“故事演员”。用户、运营和合作方都能围绕同一套故事 JSON 生产内容，后续可以扩展为精选故事广场、亲子共创、课堂创作任务和品牌活动投稿。

如果 famous-pets 跑通，MagicHaqi 还会成为“授权宠物内容入口”：用户不是简单看广告或看截图，而是在一个短故事中把角色孵化成自己的长期宠物资产。

星球定居功能则让品牌与活动入口更完整：合作方不只提供一只宠物或一个故事，还可以拥有一个对应主题母星。用户通过专属链接进入后，第一眼看到的世界氛围、引导文案、初始场景和活动入口都能围绕目标人群设计，后续仍然接入统一的宠物养成、图鉴、分享和 App 转化链路。

- “如果你也玩过那些年的儿童网页游戏，这里有一段新的童年支线。”

### 9.3 活动设计

- 抱抱龙回归季：孵化第一只抱抱龙，生成回归纪念卡。
- 我的哈奇回忆：老用户提交当年故事，优秀故事变成宠物绘本模板。
- 星球装扮赛：晒出自己的宠物房间和星球。
- 亲子创作周：孩子和家长一起生成宠物故事书。
- 限定 DNA 活动：节日限定基因、经典宠物伙伴回归。
- 经典场景互动 MV：用授权复古游戏截图和角色做 3-5 幕短故事，点击照顾宠物推进剧情，完成后获得限定 famous-pet。
- 联名宠物图鉴：合作方宠物进入“稀有宠物”页，未获得时显示问号，孵化后解锁完整阶段相册。
- 同代记忆征集：收集玩家对 2009-2012 年儿童网页游戏的共同回忆，筛选可自有化、可原创化、可授权合作的故事主题。
- IP 星球发行样板：为一个动漫 IP、课程角色或实体品牌创建专属星球，用户扫码领取专属宠物，完成故事任务后解锁优惠券、会员权益或限定家具。
- 多品牌宠物互访季：多个中小 IP 同时发行星球，用户带着自己的宠物访问不同主题星球，完成跨星球故事任务并生成多角色合影分享卡。

## 10. 风险与应对

| 风险 | 表现 | 应对 |
| --- | --- | --- |
| 怀旧只带来一次访问 | 老用户打卡后离开 | 用宠物记忆、每日成长、绘本连续剧提高复访 |
| AI 成本过高 | 免费生成被滥用 | 次数限制、模板兜底、会员扩容、排队机制 |
| Story Maker 内容质量不稳定 | AI 故事过长、对白无聊、互动节点不可玩 | 使用固定 JSON 结构、精选模板、生成后预览试玩、运营审核和模板兜底 |
| 儿童合规风险 | 聊天内容、付费、时长问题 | 内容安全过滤、家长提示、付费确认、未成年人策略 |
| App 转化不足 | H5 用户不下载 | App 必须提供 H5 没有的 3D 小镇、语音、社交或高级内容 |
| IP 信号不足 | 用户不联想到哈奇 | 首屏、宠物、文案、活动都强化抱抱龙与哈奇岛 |
| 授权素材风险 | 未授权截图、角色或名字引发版权问题 | famous-pets 只接入自有或明确授权素材，合作资料留档，活动上线前做版权检查 |
| 同期 IP 复刻冲动 | 用户希望直接复刻摩尔庄园、赛尔号等外部 IP | 明确平台规则：只做授权合作或原创致敬；提供审核、举报、下架和素材黑名单机制 |
| 联名活动像广告 | 用户觉得只是品牌露出 | 必须让互动 MV 结束后产生可养成宠物资产，并接入星球、图鉴和分享闭环 |
| IP 星球质量参差 | 中小 IP 自助配置后体验粗糙、素材不统一 | 提供标准模板、审核清单、视觉规范、预览试玩和上线审批流程 |
| B 端销售周期过长 | 品牌方认可概念但迟迟不采购 | 先用低价样板和数据报告验证，再沉淀可复制套餐、报价和行业案例 |
| 多品牌互相稀释 | 太多 IP 进入后主品牌 MagicHaqi 变弱 | 统一账号、宠物规则、星球入口和平台视觉；外部 IP 作为主题星球存在，不破坏主世界观 |
| 内容完成度不够 | TapTap 差评 | TapTap 延后到闭环更完整后，以测试版预期管理上线 |

## 11. 决策指标

5 个月后用数据决定下一阶段路径：

| 指标 | 说明 | 决策含义 |
| --- | --- | --- |
| 孵化完成率 | 新用户是否进入核心体验 | 低于 30% 需重做新手流程 |
| 次日 / 7 日留存 | 宠物养成是否成立 | 7 日低于 8% 不宜重投入 App |
| 分享系数 | 绘本传播是否成立 | 分享新用户占比低于 10% 需重做分享内容 |
| 故事生成到试玩率 | Story Maker 是否能把 AI 内容转成可玩体验 | 低于 30% 需简化创作流程或加强模板引导 |
| 互动 MV 完成率 | 授权场景故事是否足够轻、足够吸引 | 低于 40% 需缩短流程或增强奖励 |
| 稀有宠物孵化率 | famous-pets 是否能形成收集和付费动机 | 低于 5% 需调整价格、入口或宠物吸引力 |
| 同代 IP 提及率 | 用户是否自然联想到其他同期儿童网页游戏 | 如果评论、分享和问卷中高频出现，可启动授权合作 BD |
| IP 星球样板转化 | 品牌或中小 IP 是否愿意发行自己的星球 | 如果样板完成率、分享率、权益核销和复购数据成立，可产品化为 B 端套餐 |
| 私域传播系数 | IP 方自带小圈层是否能带来新增用户 | 低于预期需优化扫码入口、分享奖励和跨星球任务 |
| App 下载 / 预约转化 | H5 漏斗价值 | 低于 3% 需强化 App 独占价值 |
| 付费转化 | 商业化可行性 | 1%-3% 是早期可接受验证线 |
| AI 成本 / 用户 | 单位经济模型 | 成本必须可被会员、生成包或 App LTV 覆盖 |
| 老用户正反馈 | IP 召回强度 | 如果怀旧反馈强，继续加经典宠物与哈奇岛内容 |

## 12. 当前结论

经过挑战，最优路线是“老用户召回切入，新用户增长扩展，AI 宠物绘本传播，famous-pets 授权互动孵化验证内容合作，App / 小程序承接深度体验”。

前 5 个月不建议直接押注完整 3D App，也不建议只做纯儿童新产品。MagicHaqi 的优势在于它同时拥有老 IP 情感资产和 AI 新体验，应先用 H5 快速证明：用户愿意回来、愿意孵化、愿意分享、愿意为更深陪伴付费。

一旦 H5 数据验证成功，下一阶段可以更有信心地投入魔法哈奇手机 App、微信小程序和 TapTap 社区，并把抱抱龙、宠物伙伴、哈奇岛、famous-pets 联名宠物和 AI 陪伴做成一个可长期运营的新品牌。
